## Task 6 — Build Rate Limiter

Create a rate limiter to prevent RPC abuse when polling.

```typescript
// TODO: Implement a rate limiter class
class RateLimiter {
  private lastCall: number = 0;
  private minInterval: number;
  
  constructor(requestsPerSecond: number) {
    // TODO: Calculate minimum interval between requests
    this.minInterval = 1000 / requestsPerSecond;
  }
  
  async throttle(): Promise<void> {
    // TODO: Calculate time since last call
    const now = Date.now();
    const timeSinceLastCall = now - this.lastCall;
    
    // TODO: If not enough time has passed, wait
    if (timeSinceLastCall < this.minInterval) {
      const waitTime = this.minInterval - timeSinceLastCall;
      await new Promise(resolve => setTimeout(resolve, waitTime));
    }
    
    // TODO: Update last call time
    this.lastCall = Date.now();
  }
  
  async execute<T>(fn: () => Promise<T>): Promise<T> {
    // TODO: Apply throttle before executing function
    await this.throttle();
    return await fn();
  }
}

// Example usage:
const limiter = new RateLimiter(2); // 2 requests per second

async function testRateLimiter() {
  for (let i = 0; i < 5; i++) {
    await limiter.execute(async () => {
      console.log(`Request ${i + 1} at ${new Date().toISOString()}`);
      return getPoolPrice();
    });
  }
}
```

- Support configurable requests per second

### Validation Checklist

- ✅ SOL balance matches Solscan explorer
- ✅ USDC balance matches Solscan explorer
- ✅ Calculated price matches pool interface
- ✅ Polling bot runs without errors
- ✅ Price differences logged correctly
- ✅ Subscriptions receive updates
- ✅ Rate limiter prevents bursts

### Common Mistakes to Avoid

- ❌ **Not converting lamports to SOL** - Always divide by 10^9
- ❌ **Ignoring decimals** - USDC has 6 decimals, not 18
- ❌ **Polling too fast** - Respect RPC rate limits (2-3 req/sec max)
- ❌ **Not handling errors** - Network issues are common
- ❌ **Comparing stale prices** - Ensure timestamps are close
- ❌ **Forgetting transaction fees** - Must exceed fees to be profitable
- ❌ **Using wrong commitment** - 'confirmed' is good balance between speed and certainty
- ❌ **Hardcoding values** - Use constants for addresses
- ❌ **Not validating mint address** - Could be wrong token
- ❌ **Infinite loops without breaks** - Add stop conditions
