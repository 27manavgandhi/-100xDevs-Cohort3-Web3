## 1. ETH VS SOLANA

### Solana — Token Creation

In the browser, a `createToken` command is sent to the **Token Program** on Solana, which creates the token automatically.

#### Real-Life Example: Giant Automated Toy Factory

Think of Solana as a **giant automated toy factory**:
1. **Submit a Request**: You (the browser) send a simple form (`createToken` command) to the factory
2. **The Factory Handles It**: The factory has all the tools (the Token Program) to create your toy (token) instantly, without requiring you to know how toys are built

**Key Characteristics:**
- No need to write your own blueprints
- The tokens are created using the factory's predefined process (Token Program)
- It's faster and simpler but doesn't allow for as much customization

### Ethereum — Token Creation

On Ethereum, a **client deploys a smart contract** to create tokens like USDC, Kirat, or DOGE. Each token is its own smart contract.

#### Real-Life Example: Construction Company

Think of Ethereum as a **construction company**. If you want to build a custom house (your token), you need to:
1. **Hire an Architect**: You (the client) need to learn how to write or modify a blueprint (Solidity code for an ERC-20 contract)
2. **Submit Your Plan to the City**: You deploy the blueprint (smart contract) to Ethereum
3. **Build the House**: Once the blueprint is approved, construction begins, and your house (token) is built on your own piece of land (smart contract)

**Key Characteristics:**
- You need to **write the smart contract** (a lot of custom work!)
- The token is tied to your specific smart contract
- You have complete control but must do everything manually

### Key Differences

| | Ethereum | Solana |
|---|---|---|
| Token Creation | Write Solidity (ERC-20 contract), deploy it | Send `createToken` to the Token Program |
| Analogy | Craftsman who builds everything from scratch | Order a custom toy from a factory |
| Customization | Complete control, fully custom | Standardized, less custom |
| Complexity | High — must write and audit smart contracts | Low — just call the Token Program |
| Speed | Slower (deploy + verify) | Instant |

### Why is Solana Easier?

Because Solana has a **built-in Token Program** that already knows how to handle tokens. You just send a request (like placing an order), and it creates the token for you. On Ethereum, you have to program the logic yourself, which takes more time and effort.

## 2. How Does DEX Work

### Liquidity Pool

A **liquidity pool** is like a **money exchange booth**:
- You provide two types of tokens (like USD and Euros) to the pool
- People come to exchange one token for the other
- Every time they swap, you earn a small fee

It's a way to help people trade tokens and earn passive income, but you take some risks if token prices change a lot.

**For Example:**
You have **10,000 USDC** and **2,000 SOL**. You decide to put them in a liquidity pool.

**What happens?**

1. **You add both tokens to the pool:**
   - 10,000 USDC
   - 2,000 SOL (Assuming 1 SOL = 5 USDC, so both tokens are equal in value)

2. **People use your pool to swap tokens:**
   - If someone swaps 500 USDC for SOL, they will take some SOL from your pool and add 500 USDC to it
   - Your pool now has **10,500 USDC** and **slightly fewer SOL**

3. **You earn fees:**
   - Each time someone swaps, you earn a small fee (e.g., 0.3%)
   - If the swap fee is 0.3%, and someone trades 500 USDC, you earn **1.5 USDC**

**What's the result?**
- Your pool grows with swap fees over time
- But if the price of SOL changes a lot, the value of your tokens in the pool may vary — this is called **impermanent loss**

### Constant Product Algorithm

It's a formula that makes sure there's always enough of both tokens in a liquidity pool, no matter how much people trade.

**The formula: x * y = k**

- **x** = the amount of one token in the pool (e.g., SOL)
- **y** = the amount of the other token in the pool (e.g., USDC)
- **k** = a constant number (doesn't change)

**How Does It Work?**

Whenever someone trades one token for the other, the total product (k) must stay the same.

**Example:**
- You add **2,000 SOL** and **10,000 USDC** to the pool
- The formula becomes: `2,000 * 10,000 = 20,000,000 (k)`

Now, if someone swaps **500 USDC for SOL:**
1. **USDC increases**: The pool now has 10,500 USDC
2. **SOL decreases**: To keep the product (k = 20,000,000) constant, the pool must now have less SOL

Let's calculate how much SOL remains:
```
New SOL amount = k ÷ new USDC amount
New SOL amount = 20,000,000 ÷ 10,500 ≈ 1,904.76 SOL
```

**So:**
- The pool now has **10,500 USDC** and **1,904.76 SOL**
- The swap removed **95.24 SOL** in exchange for **500 USDC**

## 3. Impermanent Loss

Imagine you are **selling fruit in a market**. Instead of holding your fruit (tokens) in your basket, you decide to put them in a juice shop (liquidity pool). The juice shop uses your fruit to make trades between apples (ETH) and oranges (USDC).

But here's the catch: if the price of apples (ETH) goes up or down, you might end up with **less profit** than if you had just kept the apples in your basket.

This loss is called **impermanent loss**.

### How Does It Work? — Full Example

**Step 1: You deposit equal value of ETH and USDC into the pool.**
- 1 ETH = $2,000
- 2,000 USDC
- Total value: **$4,000**

Now, the pool holds:
- 1 ETH
- 2,000 USDC

And you're earning small fees when people trade between ETH and USDC.

**Step 2: ETH price changes — ETH price jumps to $8,000.**

What happens?
- People rush to buy ETH from the pool because it's cheaper there (compared to the market)
- This means they **take ETH from the pool** and add USDC instead
- Now the pool has:
  - **0.5 ETH**
  - **4,000 USDC**

### What's the Value of Your Tokens?

Your share of the pool is now:
- 0.5 ETH × $8,000 = **$4,000**
- 4,000 USDC = **$4,000**
- **Total = $8,000**

Sounds good, right? But wait...

### If You Just Held Your Tokens:

If you kept your tokens in your wallet instead of the pool, you would have:
- 1 ETH × $8,000 = $8,000
- 2,000 USDC = $2,000
- **Total = $10,000**

### What's the Loss?

By providing liquidity, you now have **$8,000 instead of $10,000**.

**You lost $2,000** compared to just holding your tokens.

This is **impermanent loss** — you lose value because the token prices changed and the pool rebalanced.

### Why Is It Called Impermanent?

- It's called **impermanent** because the loss only becomes permanent if you withdraw your tokens
- If the price of ETH goes back to $2,000, the pool will rebalance, and the loss disappears

### Why Would You Still Use Liquidity Pools?

1. **Earning fees**: You earn fees from every trade in the pool, which can offset the loss
2. **Long-term rewards**: Some platforms give extra rewards (like governance tokens) for providing liquidity

### Simple Analogy

- Imagine you lend out your apples and oranges to a juice shop
- The shop sells more apples because they're suddenly in high demand
- When you get your share back, you have fewer apples and more oranges — but the value of apples has skyrocketed!
- You might feel, "I should've just kept my apples!"

> **Takeaway**: Impermanent loss happens when the price of your tokens changes while they're in a liquidity pool. You might earn fees, but if prices swing too much, you can lose compared to just holding your tokens.

## 4. Liquid Staking

**Liquid staking** is a way to stake your cryptocurrency (lock it up to earn rewards) while still being able to use it in other activities, like trading, lending, or participating in DeFi. It's like earning interest on your savings account while still having access to the money.

### How Does It Work?

1. **You stake your cryptocurrency:**
   - Normally, when you stake tokens (e.g., ETH or SOL), they are locked, and you can't use them until the staking period ends

2. **You receive a liquid staking token (LST):**
   - Instead of locking your tokens completely, you get a **liquid staking token** in return
   - If you stake ETH → you get **stETH** (staked ETH)
   - If you stake SOL → you get **mSOL** (Marinade SOL)

3. **You can use the LST in DeFi:**
   - This token represents your staked cryptocurrency and earns staking rewards
   - You can now use this token for trading, lending, borrowing, or even providing liquidity in DeFi

### Simple Real-Life Example

Imagine you put **$100 in a fixed deposit** at a bank, and the bank gives you a **certificate** saying "This is worth $100 and earns interest." Normally, you can't use the $100 until the deposit matures. But with the certificate (like a liquid staking token), you can:
- **Sell it** if you need cash
- **Use it as collateral** to borrow money
- **Trade it** while still earning interest on the original deposit

### Key LSTs (Liquid Staking Tokens)

| Token | What You Stake | LST You Get | Platform |
|---|---|---|---|
| stETH | ETH | stETH | Lido |
| mSOL | SOL | mSOL | Marinade Finance |
| bSOL | SOL | bSOL | BlazeStake |
| JitoSOL | SOL | JitoSOL | Jito |

## 5. Creating a Token on Solana (with Raydium)

### Token Launchpad on Solana — Steps

```
A token launchpad on Solana:
1. Initialise a React project
2. Add wallet adapter to it
3. Add a polyfill that lets you access node functions
4. Figure out the calls to create a mint
5. Figure out the calls to create an ATA
6. Figure out the calls to mint a token
```

**Program**: https://github.com/raydium-io/raydium-cp-swap

If you ever want to create your own token on Solana and make it **tradeable** (let people buy and sell it):

1. Create your own token
2. Create a `liquidity pool` for your token
3. Add `liquidity` of both the tokens
4. Wait for people to trade on that pool

## Key Concepts Table

| Concept | Description |
|---|---|
| Token Program (Solana) | Built-in Solana program that handles all token creation — no custom code needed |
| ERC-20 (Ethereum) | Smart contract standard for tokens — must write and deploy Solidity code |
| Liquidity Pool | Smart contract holding two tokens for people to swap between them |
| Liquidity Provider (LP) | Person who deposits tokens into a pool and earns swap fees |
| x * y = k | Constant Product Market Maker formula — product of token quantities stays constant |
| Price Impact | How much a single trade moves the price — larger pool = smaller impact |
| Impermanent Loss | Loss from pool rebalancing when token prices diverge — "impermanent" until withdrawn |
| Swap Fee | % of each trade paid to liquidity providers (e.g., 0.3% on Raydium) |
| Raydium | Solana's leading DEX and AMM — uses CP-Swap (Constant Product) model |
| Liquid Staking | Stake tokens, receive an LST (e.g., mSOL) that earns rewards but stays usable |
| stETH | Liquid staking token from Lido — represents staked ETH on Ethereum |
| mSOL | Marinade SOL — liquid staking token for staked SOL on Solana |
| Token Launchpad | App that creates a mint, ATA, and mints tokens in a single React flow |
| Mint | Solana account that controls a token's supply |
| ATA | Associated Token Account — holds a user's balance of a specific token |
| createToken | Solana instruction to create a new Mint account via Token Program |
