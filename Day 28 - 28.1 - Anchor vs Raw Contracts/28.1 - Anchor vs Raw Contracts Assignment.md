## Core Assignment — Build Calculator in Both Native and Anchor

In this assignment, you'll build the same calculator smart contract twice—once using raw Solana (native) and once using Anchor framework. This hands-on comparison will help you understand what Anchor abstracts away and when to use each approach.

**Prerequisites:**
- Rust installed
- Solana CLI installed
- Anchor CLI installed (https://www.anchor-lang.com/docs/installation)
- Node.js and Bun runtime
- Basic Rust knowledge

## Testing Guide

### Native Contract Testing

```bash
# Build the program
cargo build-bpf

# Copy .so file to client directory
cp target/deploy/native_calculator.so client/

# Run tests
cd client
bun test
```

### Anchor Contract Testing

```bash
# Build
anchor build

# Run tests (starts local validator)
anchor test

# Or test against devnet
anchor test --provider.cluster devnet
```

### Common Mistakes to Avoid

- ❌ Forgetting to update program ID after `anchor build`
- ❌ Not signing with required keypairs
- ❌ Wrong account space calculation (forget 8-byte discriminator in Anchor)
- ❌ Using basic arithmetic instead of checked_* methods
- ❌ Not handling account initialization properly
- ❌ Incorrect instruction data encoding in native tests
- ❌ Missing `#[account(mut)]` for accounts that need to be modified
- ❌ Forgetting system_program in Initialize accounts
