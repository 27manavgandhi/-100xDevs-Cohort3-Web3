## Core Assignment — Build Your Own Telegram Wallet Bot

In this assignment, you'll build a fully functional **Telegram wallet bot** that allows users to manage Solana wallets, check balances, view transaction history, and send SOL—all through Telegram's interface.

**Development Environment**: Use **Node.js/TypeScript** with **Telegraf** framework and **@solana/web3.js**

## Task 1 — Set Up Bot Infrastructure

Create a new Telegram bot and set up the development environment.

### Requirements:

1. Create a new bot using BotFather
2. Save the bot token securely in a `.env` file
3. Initialize a Node.js/TypeScript project
4. Install required dependencies
5. Create a basic bot that responds to `/start`

**Starter Code:**

```typescript
// index.ts
import { Telegraf, Markup, Context } from 'telegraf';
import dotenv from 'dotenv';

dotenv.config();

const BOT_TOKEN = process.env.BOT_TOKEN;

if (!BOT_TOKEN) {
  throw new Error('BOT_TOKEN must be provided in .env file');
}

const bot = new Telegraf(BOT_TOKEN);

bot.start(async (ctx) => {
  // TODO: Implement welcome message with inline keyboard
  ctx.reply('Welcome! Bot is running.');
});

async function startBot(): Promise<void> {
  try {
    await bot.launch({
      allowedUpdates: ['message', 'callback_query']
    });
    console.log('✅ Bot started successfully!');
  } catch (error) {
    console.error('❌ Failed to start bot:', error);
    process.exit(1);
  }
}

startBot();

// Enable graceful stop
process.once('SIGINT', () => bot.stop('SIGINT'));
process.once('SIGTERM', () => bot.stop('SIGTERM'));
```

**Dependencies to Install:**
```bash
npm install telegraf @solana/web3.js dotenv
npm install -D typescript @types/node ts-node
```

## Testing Guide

### Setting Up Test Environment

1. **Get Devnet SOL:**
   ```bash
   solana airdrop 2 <YOUR_WALLET_ADDRESS> --url devnet
   ```
   Or use: https://solfaucet.com/

2. **Test Wallet Generation:**
   - Click "Generate Wallet"
   - Verify public key appears
   - Export private key and re-import to verify

3. **Test Balance Checking:**
   - Check balance of new wallet (should be 0)
   - Airdrop some devnet SOL
   - Check balance again (should show airdropped amount)

4. **Test Transaction Sending:**
   - Create two wallets
   - Airdrop SOL to first wallet
   - Send from first to second
   - Verify both balances update correctly

5. **Test Error Cases:**
   - Try sending with 0 balance
   - Try sending to invalid address
   - Try sending more than balance
   - Verify error messages are user-friendly

### Common Mistakes to Avoid

- ❌ **Forgetting to call `ctx.answerCbQuery()`** → Infinite loading spinner
- ❌ **Not validating Solana addresses** → Transaction failures
- ❌ **Using mainnet for testing** → Loss of real money!
- ❌ **Storing private keys in plain text** → Major security vulnerability
- ❌ **Not handling RPC errors** → Bot crashes on network issues
- ❌ **Forgetting to convert SOL to lamports** → Sending wrong amounts
- ❌ **Not checking balance before sending** → Transaction failures
- ❌ **Hardcoding bot token in code** → Token leaks in git repos
