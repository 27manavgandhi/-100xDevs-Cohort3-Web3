## Testing Guide

### Testing ERC-20 Tokens in Remix

1. **Deploy token contract**
   - Deploy with initial parameters (name, symbol, initial supply)

2. **Test basic operations**
   - Check balanceOf deployer
   - Transfer tokens to another address
   - Verify balances updated

3. **Test allowances**
   - Alice approves Bob for 100 tokens
   - Bob calls transferFrom
   - Check allowance decreased

4. **Test minting/burning**
   - Owner mints new tokens
   - User burns their tokens
   - Verify totalSupply changed

5. **Test edge cases**
   - Try to transfer more than balance (should fail)
   - Try to transferFrom without allowance (should fail)
   - Non-owner tries to mint (should fail)

### Common Mistakes to Avoid

- ❌ Forgetting to update totalSupply when minting/burning
- ❌ Not checking balance before transfer
- ❌ Not decreasing allowance in transferFrom
- ❌ Forgetting to emit events
- ❌ Using wrong decimal places (1 token = 1 * 10^18 wei)
- ❌ Not using SafeMath (though Solidity 0.8+ has built-in overflow protection)
