## 2. Writing a One-Way Bridge USDT Program

Let's start by building a complete client-side program for our one-way bridge.

### Contract Architecture

We have two contracts deployed:

**Chain 1 (Ethereum):**
```solidity
contract BridgeETH is Ownable {
    uint256 public balance;
    address public tokenAddress;
    
    event Deposit(address indexed depositor, uint amount);
    
    mapping(address => uint256) public pendingBalance;
    
    constructor(address _tokenAddress) Ownable(msg.sender) {
        tokenAddress = _tokenAddress;
    }
    
    function deposit(IERC20 _tokenAddress, uint256 _amount) public {
        require(_tokenAddress.allowance(msg.sender, address(this)) >= _amount);
        require(_tokenAddress.transferFrom(msg.sender, address(this), _amount));
        pendingBalance[msg.sender] += _amount;
        emit Deposit(msg.sender, _amount);
    }
    
    function withdraw(IERC20 _tokenAddress, uint256 _amount) public {
        require(pendingBalance[msg.sender] >= _amount);
        pendingBalance[msg.sender] -= _amount;
        _tokenAddress.transfer(msg.sender, _amount);
    }
}
```

**Chain 2 (Base/Sepolia):**
```solidity
interface IBUSDT is IERC20 {
    function mint(address _to, uint256 _amount) external;
    function burn(address _from, uint256 _amount) external;
}

contract BridgeETH is Ownable {
    uint256 public balance;
    address public tokenAddress;
    
    event Burn(address indexed burner, uint amount);
    
    mapping(address => uint256) public pendingBalance;
    
    constructor(address _tokenAddress) Ownable(msg.sender) {
        tokenAddress = _tokenAddress;
    }
    
    function burn(IBUSDT _tokenAddress, uint256 _amount) public {
        _tokenAddress.burn(msg.sender, _amount);
        emit Burn(msg.sender, _amount);
    }
    
    function withdraw(IBUSDT _tokenAddress, uint256 _amount) public {
        require(pendingBalance[msg.sender] >= _amount);
        pendingBalance[msg.sender] -= _amount;
        _tokenAddress.mint(msg.sender, _amount);
    }
}
```

### Real-Life Analogy

Think of this like a **currency exchange service**:
- **Ethereum** = US bank (you deposit dollars)
- **Base** = Foreign bank (you receive euros)
- **Our program** = The exchange service that monitors both banks

### Function to Poll Block and Get Transfers

```typescript
import { JsonRpcProvider, id } from 'ethers';
import { abi } from './abi';

const provider = new JsonRpcProvider(
  "https://eth-mainnet.g.alchemycom/v2/rnvh0pKQIYsJvNb"
);

async function pollBlock(blockNumber: number) {
  const logs = await provider.getLogs({
    address: "0xdac17f958d2ee523a2206206994597c13d831ec7",
    fromBlock: blockNumber,
    toBlock: blockNumber + 2,
    topics: [id("Transfer(address,address,uint256)")]
  });
  
  console.log(logs);
}

pollBlock(21493826);
```

### Understanding the Code

| Component | Purpose |
|-----------|---------|
| **JsonRpcProvider** | Connects to Ethereum network via RPC |
| **getLogs()** | Retrieves event logs from blockchain |
| **address** | Contract address to monitor |
| **fromBlock/toBlock** | Range of blocks to search |
| **topics** | Event signature to filter (Transfer event) |
| **id()** | Hashes event signature to topic hash |

## 4. Writing to a Contract (From the Browser)

The **useWriteContract** hook allows you to mutate data on a smart contract from a payable or nonpayable (write) function.

### Create AllowUSDT Component

```typescript
import * as React from 'react'
import { useWriteContract } from 'wagmi'

export function AllowUSDT() {
  const { data: hash, writeContract } = useWriteContract()

  async function submit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault(Default);
    writeContract({
      address: '0xdac17f958d2ee523a2206206994597c13d831ec7',
      abi: [
        {
          "constant": false,
          "inputs": [
            {
              "name": "_spender",
              "type": "address"
            },
            {
              "name": "_value",
              "type": "uint256"
            }
          ],
          "name": "approve",
          "outputs": [],
          "payable": false,
          "stateMutability": "nonpayable",
          "type": "function"
        }
      ],
      functionName: 'approve',
      args: ["0xe7b6d7305b0a2702d0f8c493c9c5e7138313ec7", BigInt(1)]
    })
  }

  return (
    <form onSubmit={submit}>
      <input name="tokenId" placeholder="69420" required />
      <button type="submit">Approve</button>
      {hash && <div>Transaction Hash: {hash}</div>}
    </form>
  )
}
```

### What This Does

```
User fills form → Clicks submit → writeContract() is called
                                           ↓
                               MetaMask pops up asking for approval
                                           ↓
                               User confirms transaction
                                           ↓
                               Transaction is sent to blockchain
                                           ↓
                               Hash is returned and displayed
```

### Key Points

- **writeContract** is the main function to send transactions
- **abi** defines the function signature
- **args** are the function parameters
- **hash** is returned after transaction submission
- User must approve transaction in MetaMask

## 5. Calling Read-Only Functions (From the Browser)

The **useReadContract** hook allows you to read data from a smart contract.

### Setup

First, get the ABI of the contract. For the USDT contract, the ABI looks like this:

```json

    "inputs": [
```

#### 6. Create wallet-options.tsx

```typescript
import * as React from 'react'
import { Connector, useConnect } from 'wagmi'

export function WalletOptions() {
  const { connectors, connect } = useConnect()

  return connectors.map((connector) => (
    <button key={connector.uid} onClick={() => connect({ connector })}>
      {connector.name}
    </button>
  ))
}
```

#### 7. Create Account.tsx

```typescript
import { useAccount, useDisconnect, useEnsAvatar, useEnsName } from 'wagmi'

export function Account() {
  const { address } = useAccount()
  const { disconnect } = useDisconnect()
  const { data: ensName } = useEnsName({ address })
  const { data: ensAvatar } = useEnsAvatar({ name: ensName! })

  return (
    <div>
      {ensAvatar && <img alt="ENS Avatar" src={ensAvatar} />}
      {address && <div>{ensName ? `${ensName} (${address})` : address}</div>}
      <button onClick={() => disconnect()}>Disconnect</button>
    </div>
  )
}
```

#### 8. Update App.tsx

```typescript
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import { WagmiProvider, useAccount } from 'wagmi'
import { config } from './config'
import { Account } from './Account'
import { WalletOptions } from './wallet-options'

const queryClient = new QueryClient()

function ConnectWallet() {
  const { isConnected } = useAccount()
  if (isConnected) return <Account />
  return <WalletOptions />
}

function App() {
  return (
    <WagmiProvider config={config}>
      <QueryClientProvider client={queryClient}>
        <ConnectWallet />
      </QueryClientProvider>
    </WagmiProvider>
  )
}

export default App
```

## 7. ETH RPC

To interact with an ETH RPC, you can send a request via CURL/POSTMAN directly to a contract.

For example, for the USDT contract:  
`https://etherscan.io/address/0xdac17f958d2ee523a2206206994597c13d831ec7`

### Total Balance

Calling read-only function `totalBalance`:

```bash
curl \
  -X POST \
  -H "Content-Type: application/json" \
  --data '{
    "jsonrpc": "2.0",
    "method": "eth_call",
    "params": [
      {
        "to": "0xdac17f958d2ee523a2206206994597c13d831ec7",
        "data": "0x18160ddd"
      },
      "latest"
    ],
    "id": 1
  }' \
  <YOUR_RPC_ENDPOINT>
```

**`data`** is the first **4 bytes** of the **keccak_256** hash of the function name (`totalSupply`)

Ref: https://emn178.github.io/online-tools/keccak_256.html

### Balance Of

**`balanceOf`** requires an argument (the address of the user whose balance you want). To find `data`:

1. Get first 4 bytes of hash of `balanceOf(address)`
2. Append the arguments after that (for eg. `0x5BFEaEcf308baB27950a352A79f8aB2795ca1A5A2`). You need to first convert this to 32 bytes before appending:
   - `0x5BFEaEcf308baB27950a352A79f8aB2795ca1A5A2` ==
   - `00000000000000000000000005BFEaEcf308baB27950a352A79f8aB2795ca1A5A2`

3. Final: 
   - `data` = `0x70a082310000000000000000000000005BFEaEcf308baB2795ca1A5A270f8c1dFaf9e3f`

```bash
curl \
  -X POST \
  -H "Content-Type: application/json" \
  --data '{
    "jsonrpc": "2.0",
    "method": "eth_call",
    "params": [
      {
        "to": "0xdac17f958d2ee523a2206206994597c13d831ec7",
        "data": "0x70a08231000000000000000000000000005BFEaEcf308baB2795ca1A5A270f8c1dFaf9e3f"
      },
      "latest"
    ],
    "id": 1
  }' \
  <YOUR_RPC_ENDPOINT>
```

## Summary

Today we covered complete client-side Ethereum development:

1. **Server-Side (Node.js)**: Used ethers.js to monitor events and interact with contracts
2. **Client-Side (React)**: Used Wagmi hooks for reading and writing to contracts
3. **Wallet Integration**: Connected MetaMask using wagmi connectors
4. **RPC Calls**: Made direct HTTP calls to Ethereum nodes
5. **Event Monitoring**: Listened to blockchain events in real-time

The key is understanding that:
- **Read operations** are free and instant (useReadContract)
- **Write operations** cost gas and require wallet confirmation (useWriteContract)
- **Events** allow us to monitor contract activity
- **ABIs** define how to interact with contracts
