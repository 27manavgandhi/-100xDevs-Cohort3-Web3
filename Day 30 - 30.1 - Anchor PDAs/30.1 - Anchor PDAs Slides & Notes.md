## 1. What are PDAs?

**Program Derived Address (PDA)** is a special type of account address that:
- Is **derived deterministically** from seeds and a program ID
- Does **not have a corresponding private key**
- Can **only be signed by the program** that derived it

### Real-Life Analogy

Think of PDAs like **safe deposit boxes**:

- **Regular Account**: You have a key (private key) to your box
- **PDA Account**: The bank (program) has the master key, derived from your name + box number (seeds)
  - Only the bank can open it
  - But the box number is deterministic: "John Doe" + "Box 42" = always the same box
  - Anyone can find which box is yours, but only the bank can open it

```
┌────────────────────────────────────────────────────┐
│           Regular Account vs PDA                    │
├────────────────────────────────────────────────────┤
│                                                     │
│  Regular Account (Keypair):                        │
│  ┌─────────────────────────────────────────┐      │
│  │  Private Key ──→ Public Key             │      │
│  │                                          │      │
│  │  User controls via signature             │      │
│  │  Can be used anywhere                    │      │
│  └─────────────────────────────────────────┘      │
│                                                     │
│  Program Derived Address (PDA):                    │
│  ┌─────────────────────────────────────────┐      │
│  │  Seeds + Program ID ──→ Address         │      │
│  │                                          │      │
│  │  NO private key exists                   │      │
│  │  Only program can sign (invoke_signed)   │      │
│  │  Deterministic (same seeds = same PDA)   │      │
│  └─────────────────────────────────────────┘      │
│                                                     │
└────────────────────────────────────────────────────┘
```

### PDA Characteristics

| Characteristic | Description |
|----------------|-------------|
| **Deterministic** | Same seeds + program = same address every time |
| **No Private Key** | Cannot sign transactions like normal accounts |
| **Program Signing** | Only the deriving program can sign for it |
| **Unique per Program** | Same seeds in different programs = different PDAs |
| **Off the Curve** | Not on Ed25519 elliptic curve (no valid keypair) |

## 2. Why Use PDAs?

### Use Cases

1. **Deterministic Accounts**
   - Always find user's account without storing addresses
   - Example: Token vault for user → seeds: [b"vault", user.key()]

2. **Program-Controlled Accounts**
   - Program manages funds/assets on behalf of users
   - Example: Escrow, vaults, treasuries

3. **Mapping/Association**
   - Associate data with specific entities
   - Example: User → Staking Account, NFT → Metadata

4. **Singleton Accounts**
   - Single global account per program
   - Example: Global config → seeds: [b"config"]

### Benefits

```
Traditional Approach (Keypairs):
  User creates account → stores address → passes to program
  Problems:
    - User must remember/store address
    - No guarantee address is correct
    - Extra storage/lookup needed

PDA Approach:
  Program derives address → always same → no storage needed
  Benefits:
    ✓ Deterministic (always find it)
    ✓ Secure (only program controls)
    ✓ No extra storage
    ✓ Canonical address
```

## 3. PDA Derivation

### How PDAs are Created

```rust
// Find PDA with seeds
let (pda, bump) = Pubkey::find_program_address(
    &[
        b"vault",              // Static seed
        user.key().as_ref(),   // Dynamic seed (user's pubkey)
    ],
    program_id,
);
```

**Process:**
1. Start with seeds (bytes arrays)
2. Add program ID
3. Hash using SHA256
4. Check if result is on Ed25519 curve
5. If on curve, increment bump and try again
6. If off curve, that's your PDA!

### The "Bump" Seed

The **bump** is a single byte (255 → 0) used to find an address **off the curve**.

```
Try bump = 255:
  hash([seeds..., program_id, 255])
  → On curve? Try 254
  
Try bump = 254:
  hash([seeds..., program_id, 254])
  → Off curve! Found PDA
  
PDA = address with bump 254
Canonical bump = 254 (first one that works)
```

**Why off the curve?**
- Addresses on the curve have private keys
- PDAs must NOT have private keys
- Off-curve = no valid keypair = only program can sign

## 4. PDAs in Native Solana

### Deriving a PDA

```rust
use solana_program::{
    pubkey::Pubkey,
    program::invoke_signed,
};

// Find PDA
let user_pubkey = /* user's address */;
let seeds = &[
    b"user_account",
    user_pubkey.as_ref(),
];

let (pda, bump) = Pubkey::find_program_address(seeds, program_id);
```

### Creating Account at PDA

```rust
// Create account at PDA using CPI with invoke_signed
let pda_seeds: &[&[u8]] = &[
    b"user_account",
    user_pubkey.as_ref(),
    &[bump],  // Include bump
];

let create_account_ix = system_instruction::create_account(
    payer.key,
    &pda,
    lamports,
    space as u64,
    program_id,
);

// invoke_signed allows program to sign on behalf of PDA
invoke_signed(
    &create_account_ix,
    &[payer.clone(), pda_account.clone(), system_program.clone()],
    &[pda_seeds],  // Signer seeds
)?;
```

### Key Difference: invoke_signed

```rust
// invoke() - normal CPI
invoke(
    &instruction,
    &[account1, account2],
)?;

// invoke_signed() - CPI with PDA signing
invoke_signed(
    &instruction,
    &[account1, account2],
    &[signer_seeds],  // Program signs using these seeds
)?;
```

## 5. PDAs in Anchor

Anchor makes PDAs dramatically simpler with `seeds` and `bump` constraints.

### Deriving PDA in Anchor

```rust
#[derive(Accounts)]
pub struct Initialize<'info> {
    #[account(
        init,
        payer = user,
        space = 8 + 32 + 8,
        seeds = [b"user_account", user.key().as_ref()],
        bump
    )]
    pub user_account: Account<'info, UserAccount>,
    
    #[account(mut)]
    pub user: Signer<'info>,
    
    pub system_program: Program<'info, System>,
}

#[account]
pub struct UserAccount {
    pub owner: Pubkey,     // 32 bytes
    pub balance: u64,      // 8 bytes
}
```

**What Anchor does:**
1. Derives PDA from seeds
2. Finds canonical bump
3. Creates account at PDA address
4. Stores bump in account (optional)
5. Validates PDA matches expected address

### Accessing PDA in Later Instructions

```rust
#[derive(Accounts)]
pub struct Update<'info> {
    #[account(
        mut,
        seeds = [b"user_account", user.key().as_ref()],
        bump
    )]
    pub user_account: Account<'info, UserAccount>,
    
    pub user: Signer<'info>,
}
```

Anchor:
- Re-derives PDA from seeds
- Verifies provided account matches derived address
- Fails if mismatch (security check)

## 6. CPI with PDAs (invoke_signed)

PDAs can sign CPIs using `invoke_signed` or Anchor's `CpiContext::new_with_signer`.

### Native Approach

```rust
// Transfer SOL from PDA to user
let pda_seeds: &[&[u8]] = &[
    b"vault",
    user.key().as_ref(),
    &[bump],
];

let transfer_ix = system_instruction::transfer(
    pda_account.key,   // From: PDA
    user.key,          // To: User
    amount,
);

invoke_signed(
    &transfer_ix,
    &[pda_account.clone(), user.clone(), system_program.clone()],
    &[pda_seeds],  // PDA signs!
)?;
```

### Anchor Approach

```rust
use anchor_lang::system_program::{transfer, Transfer};

pub fn withdraw(ctx: Context<Withdraw>, amount: u64, bump: u8) -> Result<()> {
    let user_key = ctx.accounts.user.key();
    let seeds = &[
        b"vault",
        user_key.as_ref(),
        &[bump],
    ];
    let signer_seeds = &[&seeds[..]];
    
    // Transfer with PDA signing
    transfer(
        CpiContext::new_with_signer(
            ctx.accounts.system_program.to_account_info(),
            Transfer {
                from: ctx.accounts.vault.to_account_info(),
                to: ctx.accounts.user.to_account_info(),
            },
            signer_seeds,  // PDA signs
        ),
        amount,
    )?;
    
    Ok(())
}

#[derive(Accounts)]
#[instruction(amount: u64, bump: u8)]
pub struct Withdraw<'info> {
    #[account(
        mut,
        seeds = [b"vault", user.key().as_ref()],
        bump = bump
    )]
    pub vault: SystemAccount<'info>,
    
    #[account(mut)]
    pub user: Signer<'info>,
    
    pub system_program: Program<'info, System>,
}
```

## 7. Staking Contract Example

A complete staking program using PDAs for user accounts.

### Program Structure

```rust
use anchor_lang::prelude::*;
use anchor_lang::system_program::{transfer, Transfer};

declare_id!("YourProgramIDHere");

// Points calculation constants
const POINTS_PER_SOL_PER_DAY: u64 = 1_000_000;  // 1 point per SOL per day (micro-points)
const LAMPORTS_PER_SOL: u64 = 1_000_000_000;
const SECONDS_PER_DAY: i64 = 86_400;

#[program]
pub mod staking_program {
    use super::*;

    /// Create a PDA staking account for user
    pub fn create_pda_account(ctx: Context<CreatePdaAccount>) -> Result<()> {
        let pda_account = &mut ctx.accounts.pda_account;
        let clock = Clock::get()?;
        
        pda_account.owner = ctx.accounts.payer.key();
        pda_account.staked_amount = 0;
        pda_account.total_points = 0;
        pda_account.last_update_time = clock.unix_timestamp;
        pda_account.bump = ctx.bumps.pda_account;
        
        msg!("PDA account created successfully");
        
        Ok(())
    }
    
    /// Stake SOL into PDA account
    pub fn stake(ctx: Context<Stake>, amount: u64) -> Result<()> {
        require!(amount > 0, StakeError::InvalidAmount);
        
        let pda_account = &mut ctx.accounts.pda_account;
        let clock = Clock::get()?;
        
        // Update points before changing staked amount
        update_points(pda_account, clock.unix_timestamp)?;
        
        // Transfer SOL from user to PDA
        let cpi_context = CpiContext::new(
            ctx.accounts.system_program.to_account_info(),
            Transfer {
                from: ctx.accounts.user.to_account_info(),
                to: ctx.accounts.pda_account.to_account_info(),
            },
        );
        transfer(cpi_context, amount)?;
        
        // Update staked amount
        pda_account.staked_amount = pda_account.staked_amount
            .checked_add(amount)
            .ok_or(StakeError::Overflow)?;
        
        msg!(
            "Staked {} lamports. Total staked: {}, Total points: {}",
            amount,
            pda_account.staked_amount,
            pda_account.total_points / 1_000_000
        );
        
        Ok(())
    }
    
    /// Unstake SOL from PDA back to user
    pub fn unstake(ctx: Context<Unstake>, amount: u64) -> Result<()> {
        require!(amount > 0, StakeError::InvalidAmount);
        
        let pda_account = &mut ctx.accounts.pda_account;
        let clock = Clock::get()?;
        
        require!(
            pda_account.staked_amount >= amount,
            StakeError::InsufficientStake
        );
        
        // Update points before changing staked amount
        update_points(pda_account, clock.unix_timestamp)?;
        
        // Transfer SOL from PDA to user
        let user_key = ctx.accounts.user.key();
        let seeds = &[
            b"client",
            user_key.as_ref(),
            &[pda_account.bump],
        ];
        let signer_seeds = &[&seeds[..]];
        
        let cpi_context = CpiContext::new_with_signer(
            ctx.accounts.system_program.to_account_info(),
            Transfer {
                from: ctx.accounts.pda_account.to_account_info(),
                to: ctx.accounts.user.to_account_info(),
            },
            signer_seeds,
        );
        transfer(cpi_context, amount)?;
        
        // Update staked amount
        pda_account.staked_amount = pda_account.staked_amount
            .checked_sub(amount)
            .ok_or(StakeError::Underflow)?;
        
        msg!(
            "Unstaked {} lamports. Remaining staked: {}, Total points: {}",
            amount,
            pda_account.staked_amount,
            pda_account.total_points / 1_000_000
        );
        
        Ok(())
    }
    
    /// Claim accumulated points
    pub fn claim_points(ctx: Context<ClaimPoints>) -> Result<()> {
        let pda_account = &mut ctx.accounts.pda_account;
        let clock = Clock::get()?;
        
        // Update points to current time
        update_points(pda_account, clock.unix_timestamp)?;
        
        let claimable_points = pda_account.total_points / 1_000_000;
        
        msg!(
            "User has {} claimable points",
            claimable_points
        );
        
        // Reset points after claiming (or you could track claimed separately)
        pda_account.total_points = 0;
        
        Ok(())
    }
    
    /// Get current points without updating
    pub fn get_points(ctx: Context<GetPoints>) -> Result<()> {
        let pda_account = &ctx.accounts.pda_account;
        let clock = Clock::get()?;
        
        // Calculate current points without updating
        let time_elapsed = clock.unix_timestamp
            .checked_sub(pda_account.last_update_time)
            .ok_or(StakeError::InvalidTimestamp)? as u64;
        
        let new_points = calculate_points_earned(
            pda_account.staked_amount,
            time_elapsed
        )?;
        
        let current_total_points = pda_account.total_points
            .checked_add(new_points)
            .ok_or(StakeError::Overflow)?;
        
        msg!(
            "Current points: {}, Staked amount: {} SOL",
            current_total_points / 1_000_000,
            pda_account.staked_amount / LAMPORTS_PER_SOL
        );
        
        Ok(())
    }
}

// Helper function to update points
fn update_points(pda_account: &mut StakeAccount, current_time: i64) -> Result<()> {
    let time_elapsed = current_time
        .checked_sub(pda_account.last_update_time)
        .ok_or(StakeError::InvalidTimestamp)? as u64;
    
    if time_elapsed > 0 && pda_account.staked_amount > 0 {
        let new_points = calculate_points_earned(
            pda_account.staked_amount,
            time_elapsed
        )?;
        
        pda_account.total_points = pda_account.total_points
            .checked_add(new_points)
            .ok_or(StakeError::Overflow)?;
    }
    
    pda_account.last_update_time = current_time;
    Ok(())
}

// Calculate points earned based on staked amount and time
fn calculate_points_earned(staked_amount_lamports: u64, time_elapsed_seconds: u64) -> Result<u64> {
    // Points = (staked_amount_in_sol) * (time_in_days) * POINTS_PER_SOL_PER_DAY
    // Using micro-points for precision
    
    let points = (staked_amount_lamports as u128)
        .checked_mul(time_elapsed_seconds as u128)
        .ok_or(StakeError::Overflow)?
        .checked_mul(POINTS_PER_SOL_PER_DAY as u128)
        .ok_or(StakeError::Overflow)?
        .checked_div(LAMPORTS_PER_SOL as u128)
        .ok_or(StakeError::Overflow)?
        .checked_div(SECONDS_PER_DAY as u128)
        .ok_or(StakeError::Overflow)? as u64;
    
    Ok(points)
}

// Account Structures

#[account]
pub struct StakeAccount {
    pub owner: Pubkey,              // 32 bytes
    pub staked_amount: u64,         // 8 bytes
    pub total_points: u64,          // 8 bytes (micro-points for precision)
    pub last_update_time: i64,      // 8 bytes
    pub bump: u8,                   // 1 byte
}

// Context Structures

#[derive(Accounts)]
pub struct CreatePdaAccount<'info> {
    #[account(mut)]
    pub payer: Signer<'info>,
    
    #[account(
        init,
        payer = payer,
        space = 8 + 32 + 8 + 8 + 8 + 1,
        seeds = [b"client", payer.key().as_ref()],
        bump
    )]
    pub pda_account: Account<'info, StakeAccount>,
    
    pub system_program: Program<'info, System>,
}

#[derive(Accounts)]
pub struct Stake<'info> {
    #[account(
        mut,
        seeds = [b"client", user.key().as_ref()],
        bump = pda_account.bump,
        constraint = pda_account.owner == user.key() @ StakeError::Unauthorized
    )]
    pub pda_account: Account<'info, StakeAccount>,
    
    #[account(mut)]
    pub user: Signer<'info>,
    
    pub system_program: Program<'info, System>,
}

#[derive(Accounts)]
pub struct Unstake<'info> {
    #[account(
        mut,
        seeds = [b"client", user.key().as_ref()],
        bump = pda_account.bump,
        constraint = pda_account.owner == user.key() @ StakeError::Unauthorized
    )]
    pub pda_account: Account<'info, StakeAccount>,
    
    #[account(mut)]
    pub user: Signer<'info>,
    
    pub system_program: Program<'info, System>,
}

#[derive(Accounts)]
pub struct ClaimPoints<'info> {
    #[account(
        mut,
        seeds = [b"client", user.key().as_ref()],
        bump = pda_account.bump,
        constraint = pda_account.owner == user.key() @ StakeError::Unauthorized
    )]
    pub pda_account: Account<'info, StakeAccount>,
    
    pub user: Signer<'info>,
}

#[derive(Accounts)]
pub struct GetPoints<'info> {
    #[account(
        seeds = [b"client", user.key().as_ref()],
        bump = pda_account.bump
    )]
    pub pda_account: Account<'info, StakeAccount>,
    
    pub user: Signer<'info>,
}

// Errors

#[error_code]
pub enum StakeError {
    #[msg("Amount must be greater than 0")]
    InvalidAmount,
    #[msg("Insufficient staked amount")]
    InsufficientStake,
    #[msg("Unauthorized access")]
    Unauthorized,
    #[msg("Arithmetic overflow")]
    Overflow,
    #[msg("Arithmetic underflow")]
    Underflow,
    #[msg("Invalid timestamp")]
    InvalidTimestamp,
}
```

## Summary

Program Derived Addresses (PDAs) are fundamental to Solana development:

1. **Deterministic addresses**: Always find accounts without storage
2. **Program signing**: Only the program can sign for PDAs
3. **Secure vaults**: User funds controlled by program logic
4. **Anchor simplifies**: seeds/bump constraints handle complexity
5. **invoke_signed**: Enables PDAs to sign CPIs
6. **Real-world patterns**: Staking, escrow, vaults all use PDAs

PDAs transform Solana from simple programs into sophisticated financial applications with **deterministic, secure, program-controlled accounts**.

## Key Concepts Table

| Concept | Description |
|---------|-------------|
| PDA | Program Derived Address - deterministic address from seeds |
| Seeds | Byte arrays used to derive PDA (up to 32 bytes each) |
| Bump | Single byte to find address off Ed25519 curve |
| Canonical Bump | First bump (255→0) that produces valid PDA |
| find_program_address | Function to derive PDA from seeds |
| invoke_signed | CPI where program signs on behalf of PDA |
| Off the Curve | Address not on Ed25519 curve (no private key) |
| Signer Seeds | Seeds used by program to sign for PDA |
| Deterministic | Same seeds always produce same address |
| PDA Constraint | Anchor validation ensuring correct PDA derivation |
