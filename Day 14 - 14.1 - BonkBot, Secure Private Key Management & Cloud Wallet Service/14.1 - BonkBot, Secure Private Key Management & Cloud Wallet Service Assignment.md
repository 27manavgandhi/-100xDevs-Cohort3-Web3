## Core Assignment — Build the Cloud Wallet Service

Build a full **Cloud Wallet** backend in Node.js + TypeScript with a React frontend.

Users can sign up to get a Solana wallet, log in, and ask the server to sign & broadcast transactions on their behalf — without ever seeing their own private key.

### Frontend Setup (React + Vite)

```bash
npm create vite@latest cloud-wallet-frontend -- --template react-ts
cd cloud-wallet-frontend
npm install axios
npm install -D vite-plugin-node-polyfills
```

Update `vite.config.ts`:
```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { nodePolyfills } from 'vite-plugin-node-polyfills'

export default defineConfig({
  plugins: [
    react(),
    nodePolyfills(), // needed for Buffer, crypto in browser
  ],
})
```

**Build a UI with:**
- Signup form → shows user's public key after creating wallet
- Signin form → stores JWT in localStorage
- Send Transaction form:
  - Inputs: recipient address, amount in SOL
  - Builds transaction client-side with `@solana/web3.js`
  - Serializes with `tx.serialize({ requireAllSignatures: false })`
  - Sends base64 string to backend `/api/v1/txn/sign`
  - Polls `/api/v1/txn?id=<id>` for status

## Conceptual Questions

1. Why does a cloud wallet store private keys server-side? What are the security risks compared to a self-custody wallet?
2. What is the difference between **AES-256 encryption** and **hashing** (bcrypt)? Why do we use each for different data?
3. Why must we serialize a transaction before sending it to the backend? What does `tx.serialize()` produce?
4. What does `requireAllSignatures: false` mean when serializing a Solana transaction?
5. How does a KMS (like AWS KMS) differ from simply encrypting with AES-256 and storing the encrypted key?
6. What is the `retry` flag in the sign txn request body? When would a client set it to `true`?
7. How would you prevent a compromised JWT from being used to drain a user's wallet?

### Challenge 1 — Real Database

Replace in-memory storage with a real database (PostgreSQL + Prisma or SQLite):

```bash
npm install prisma @prisma/client
npx prisma init
```

Schema:
```prisma
model User {
  id                   String        @id @default(cuid())
  username             String        @unique
  passwordHash         String
  encryptedPrivateKey  String
  publicKey            String
  transactions         Transaction[]
}

model Transaction {
  id         String   @id @default(cuid())
  userId     String
  txSignature String?
  status     String   @default("processing")
  user       User     @relation(fields: [userId], references: [id])
  createdAt  DateTime @default(now())
}
```
