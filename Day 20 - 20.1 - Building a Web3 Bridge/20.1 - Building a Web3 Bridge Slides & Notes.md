## 2. Smart Contract Layer - Source Chain

### BridgeETH Contract (Ethereum/Sepolia)

The source chain contract is responsible for:
- Accepting and locking user tokens
- Tracking locked amounts
- Allowing withdrawals after burns
- Emitting events for the operator

```solidity
contract BridgeETH {
    IERC20 public token;
    mapping(address => uint256) public lockedBalance;
    mapping(bytes32 => bool) public processedWithdrawals;
    
    event TokensLocked(
        address indexed user,
        uint256 amount,
        uint256 timestamp,
        bytes32 indexed transferId
    );
    
    event TokensReleased(
        address indexed user,
        uint256 amount,
        bytes32 indexed transferId
    );
    
    function lockTokens(uint256 _amount) external returns (bytes32) {
        // Generate unique transfer ID
        bytes32 transferId = keccak256(
            abi.encodePacked(msg.sender, _amount, block.timestamp, block.number)
        );
        
        // Transfer tokens from user to bridge
        token.transferFrom(msg.sender, address(this), _amount);
        
        // Update locked balance
        lockedBalance[msg.sender] += _amount;
        
        // Emit event for bridge operator
        emit TokensLocked(msg.sender, _amount, block.timestamp, transferId);
        
        return transferId;
    }
    
    function releaseTokens(
        address _user,
        uint256 _amount,
        bytes32 _burnTxHash
    ) external onlyOperator {
        // Prevent replay attacks
        require(!processedWithdrawals[_burnTxHash], "Already processed");
        
        // Mark as processed
        processedWithdrawals[_burnTxHash] = true;
        
        // Transfer tokens back to user
        token.transfer(_user, _amount);
        
        // Update balance
        lockedBalance[_user] -= _amount;
        
        emit TokensReleased(_user, _amount, _burnTxHash);
    }
}
```

### Key Features

| Feature | Purpose |
|---------|---------|
| **Transfer ID** | Unique identifier for each bridge transaction |
| **Nonce Tracking** | Prevents processing same transaction twice |
| **Event Emission** | Notifies bridge operator of state changes |
| **Access Control** | Only operator can trigger releases |

## 3. Smart Contract Layer - Destination Chain

### BridgeBase Contract (Base/Base Sepolia)

The destination chain contract handles:
- Minting wrapped tokens
- Burning tokens for returns
- Event emission for monitoring

```solidity
contract BridgeBase {
    IBUSDT public wrappedToken;
    mapping(bytes32 => bool) public processedMints;
    
    event TokensMinted(
        address indexed user,
        uint256 amount,
        bytes32 indexed transferId
    );
    
    event TokensBurned(
        address indexed user,
        uint256 amount,
        uint256 timestamp,
        bytes32 indexed burnId
    );
    
    function mintTokens(
        address _user,
        uint256 _amount,
        bytes32 _transferId
    ) external onlyOperator {
        // Prevent replay
        require(!processedMints[_transferId], "Already minted");
        
        // Mark as processed
        processedMints[_transferId] = true;
        
        // Mint wrapped tokens
        wrappedToken.mint(_user, _amount);
        
        emit TokensMinted(_user, _amount, _transferId);
    }
    
    function burnTokens(uint256 _amount) external returns (bytes32) {
        // Generate burn ID
        bytes32 burnId = keccak256(
            abi.encodePacked(msg.sender, _amount, block.timestamp)
        );
        
        // Burn user's wrapped tokens
        wrappedToken.burn(msg.sender, _amount);
        
        emit TokensBurned(msg.sender, _amount, block.timestamp, burnId);
        
        return burnId;
    }
}
```

### Core Service Code Structure

```typescript
import { ethers } from 'ethers';
import mongoose from 'mongoose';

class BridgeOperator {
    private ethProvider: ethers.JsonRpcProvider;
    private baseProvider: ethers.JsonRpcProvider;
    private ethBridge: ethers.Contract;
    private baseBridge: ethers.Contract;
    private operatorWallet: ethers.Wallet;
    
    constructor() {
        // Initialize providers
        this.ethProvider = new ethers.JsonRpcProvider(ETH_RPC);
        this.baseProvider = new ethers.JsonRpcProvider(BASE_RPC);
        
        // Initialize wallet
        this.operatorWallet = new ethers.Wallet(OPERATOR_PRIVATE_KEY);
        
        // Initialize contracts
        this.ethBridge = new ethers.Contract(
            ETH_BRIDGE_ADDRESS,
            BRIDGE_ETH_ABI,
            this.operatorWallet.connect(this.ethProvider)
        );
        
        this.baseBridge = new ethers.Contract(
            BASE_BRIDGE_ADDRESS,
            BRIDGE_BASE_ABI,
            this.operatorWallet.connect(this.baseProvider)
        );
    }
    
    async start() {
        console.log('🌉 Bridge Operator Starting...');
        
        // Listen for lock events on Ethereum
        this.ethBridge.on('TokensLocked', async (user, amount, timestamp, transferId) => {
            await this.handleTokensLocked(user, amount, transferId);
        });
        
        // Listen for burn events on Base
        this.baseBridge.on('TokensBurned', async (user, amount, timestamp, burnId) => {
            await this.handleTokensBurned(user, amount, burnId);
        });
        
        console.log('✅ Bridge Operator Running');
    }
    
    async handleTokensLocked(user: string, amount: bigint, transferId: string) {
        console.log(`🔒 Tokens Locked: ${user} - ${amount}`);
        
        try {
            // Save to database
            await this.saveTransfer({
                transferId,
                user,
                amount: amount.toString(),
                status: 'locked',
                sourceChain: 'ethereum',
                destChain: 'base'
            });
            
            // Wait for confirmations
            await this.waitForConfirmations('ethereum', 3);
            
            // Mint on Base
            const tx = await this.baseBridge.mintTokens(user, amount, transferId);
            await tx.wait();
            
            console.log(`✅ Minted on Base: ${tx.hash}`);
            
            // Update status
            await this.updateTransferStatus(transferId, 'completed');
        } catch (error) {
            console.error('❌ Error processing lock:', error);
            await this.updateTransferStatus(transferId, 'failed');
        }
    }
    
    async handleTokensBurned(user: string, amount: bigint, burnId: string) {
        console.log(`🔥 Tokens Burned: ${user} - ${amount}`);
        
        try {
            // Save to database
            await this.saveBurn({
                burnId,
                user,
                amount: amount.toString(),
                status: 'burned',
                sourceChain: 'base',
                destChain: 'ethereum'
            });
            
            // Wait for confirmations
            await this.waitForConfirmations('base', 3);
            
            // Release on Ethereum
            const tx = await this.ethBridge.releaseTokens(user, amount, burnId);
            await tx.wait();
            
            console.log(`✅ Released on Ethereum: ${tx.hash}`);
            
            // Update status
            await this.updateBurnStatus(burnId, 'completed');
        } catch (error) {
            console.error('❌ Error processing burn:', error);
            await this.updateBurnStatus(burnId, 'failed');
        }
    }
}
```

## 6. Frontend Integration

### React Component for Bridge UI

```typescript
import { ethers } from 'ethers';
import { useState } from 'react';

function BridgeUI() {
    const [amount, setAmount] = useState('');
    const [status, setStatus] = useState('');
    
    async function bridgeToBase() {
        setStatus('Connecting wallet...');
        
        // Connect to MetaMask
        const provider = new ethers.BrowserProvider(window.ethereum);
        const signer = await provider.getSigner();
        
        // Get contracts
        const token = new ethers.Contract(USDT_ADDRESS, USDT_ABI, signer);
        const bridge = new ethers.Contract(BRIDGE_ADDRESS, BRIDGE_ABI, signer);
        
        setStatus('Approving tokens...');
        
        // Approve bridge to spend tokens
        const approveTx = await token.approve(BRIDGE_ADDRESS, ethers.parseUnits(amount, 6));
        await approveTx.wait();
        
        setStatus('Locking tokens...');
        
        // Lock tokens
        const lockTx = await bridge.lockTokens(ethers.parseUnits(amount, 6));
        const receipt = await lockTx.wait();
        
        // Get transfer ID from event
        const event = receipt.logs.find(log => {
            try {
                return bridge.interface.parseLog(log)?.name === 'TokensLocked';
            } catch { return false; }
        });
        
        const transferId = bridge.interface.parseLog(event).args.transferId;
        
        setStatus(`Bridging... Transfer ID: ${transferId}`);
        
        // Poll for completion
        await pollForCompletion(transferId);
        
        setStatus('✅ Bridge Complete!');
    }
    
    async function pollForCompletion(transferId: string) {
        // Call backend API to check status
        const interval = setInterval(async () => {
            const response = await fetch(`/api/transfer/${transferId}`);
            const data = await response.json();
            
            if (data.status === 'completed') {
                clearInterval(interval);
            }
        }, 5000); // Check every 5 seconds
    }
    
    return (
        <div className="bridge-container">
            <h2>Bridge USDT to Base</h2>
            <input
                type="number"
                value={amount}
                onChange={(e) => setAmount(e.target.value)}
                placeholder="Amount"
            />
            <button onClick={bridgeToBase}>
                Bridge to Base
            </button>
            <p>{status}</p>
        </div>
    );
}
```

## 7. Security Considerations

### Critical Security Measures

| Security Layer | Implementation |
|----------------|----------------|
| **Replay Protection** | Use unique transfer IDs, track processed transactions |
| **Nonce Management** | Prevent double-processing with mapping |
| **Multi-Sig Wallet** | Require multiple approvals for operator actions |
| **Rate Limiting** | Limit transfer amounts and frequency |
| **Confirmation Blocks** | Wait for multiple confirmations before processing |
| **Event Verification** | Validate all event data before processing |
| **Database Backup** | Regular backups for recovery |
| **Monitoring** | Alert on unusual patterns |

### Replay Attack Prevention

```solidity
// Track processed transactions
mapping(bytes32 => bool) public processedTransactions;

function processTransfer(bytes32 _transferId, ...) external {
    // Check if already processed
    require(!processedTransactions[_transferId], "Already processed");
    
    // Mark as processed BEFORE making external calls
    processedTransactions[_transferId] = true;
    
    // Process transfer
    // ...
}
```

## 10. Monitoring and Maintenance

### Key Metrics to Track

| Metric | Alert Threshold |
|--------|----------------|
| **Processing Time** | > 5 minutes |
| **Failed Transactions** | > 5% failure rate |
| **Locked vs Minted** | Imbalance > 1% |
| **Operator Balance** | < 0.1 ETH |
| **Unprocessed Events** | > 10 pending |

## Summary

Building a Web3 bridge involves multiple interconnected components:

1. **Smart contracts** on both source and destination chains for locking/minting
2. **Bridge operator service** that monitors events and processes transfers
3. **Database** for tracking transfers and recovery
4. **Frontend** for user interaction
5. **Security measures** to prevent attacks and ensure safety
6. **Monitoring** for operational health and incident response

The key to a successful bridge is:
- **Robust event handling** to catch all chain events
- **Proper nonce management** to prevent replays
- **Comprehensive testing** across all scenarios
- **24/7 monitoring** to catch and fix issues quickly
