## 1. SPL Token Standard

### What are SPL Tokens?

**SPL (Solana Program Library) Tokens** are the standard for fungible and non-fungible tokens on Solana, similar to ERC-20 tokens on Ethereum.

**Real-Life Analogy**: If SOL is like **cash in your wallet**, SPL tokens are like **gift cards, loyalty points, or foreign currency**. You need a special "slot" (token account) for each type of card you want to hold.

### Key Differences: SOL vs SPL Tokens

| Aspect | SOL (Native Token) | SPL Tokens |
|--------|-------------------|------------|
| **Account Type** | Regular wallet address | Associated Token Account (ATA) |
| **Storage** | Balance stored directly | Balance in separate token account |
| **Transfer Program** | SystemProgram | Token Program |
| **Account Creation** | Automatic | Must be created explicitly |
| **Rent** | Once per wallet | Per token type held |

## 2. Associated Token Accounts (ATA)

### What is an ATA?

An **Associated Token Account** is a special account that holds SPL tokens for a wallet. It's a deterministic address derived from:
- The wallet's public key
- The token's mint address

**Real-Life Analogy**: Think of your Solana wallet like a **bank**:
- Your main wallet address = Your bank (holds SOL)
- Each ATA = A different **safe deposit box** for each token type
- Box for USDC, box for BONK, box for WIF, etc.

### ATA Address Derivation

```
ATA Address = f(Wallet Public Key, Token Mint Address, Token Program ID)

Example:
Wallet: 7xJ9qJxmQ8ZHqGnxF4hP3vN2kD5tL1wY9aB6cR8eT3mK
USDC Mint: EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v
→ ATA: Gh9ZwEmdLJ8DscKNTkTqPbNwLNNBjuSzaG9Vp2KGtKJr
```

### Why ATAs?

| Benefit | Explanation |
|---------|-------------|
| **Deterministic** | Same wallet + same token = always same ATA |
| **Organized** | Each token type has its own account |
| **Secure** | Only wallet owner can control their ATAs |
| **Efficient** | Programs can predict ATA addresses |

## 4. Checking SPL Token Balance

### Step-by-Step Process

```
1. Get user's wallet public key
2. Get token mint address (e.g., USDC)
3. Calculate ATA address
4. Check if ATA exists
5. If exists, get balance
6. If not exists, balance = 0
```

### Implementation

```typescript
import { getAssociatedTokenAddress, getAccount } from '@solana/spl-token';
import { PublicKey } from '@solana/web3.js';

async function getTokenBalance(
  connection: Connection,
  walletPublicKey: PublicKey,
  tokenMintAddress: string
): Promise<number> {
  try {
    const mintPublicKey = new PublicKey(tokenMintAddress);
    
    // Calculate ATA address
    const ata = await getAssociatedTokenAddress(
      mintPublicKey,      // Token mint
      walletPublicKey     // Wallet owner
    );
    
    console.log('ATA Address:', ata.toBase58());
    
    // Get account info
    const accountInfo = await getAccount(connection, ata);
    
    // Get mint info for decimals
    const mintInfo = await getMint(connection, mintPublicKey);
    
    // Convert to human-readable amount
    const balance = Number(accountInfo.amount) / Math.pow(10, mintInfo.decimals);
    
    return balance;
    
  } catch (error) {
    // ATA doesn't exist = balance is 0
    console.log('Token account not found, balance = 0');
    return 0;
  }
}

// Usage
const usdcMint = 'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v';
const balance = await getTokenBalance(connection, userPublicKey, usdcMint);
console.log(`USDC Balance: ${balance}`);
```

### Understanding Decimals

| Token | Decimals | Raw Amount | Display Amount |
|-------|----------|------------|----------------|
| USDC | 6 | 1,000,000 | 1.000000 |
| BONK | 5 | 100,000 | 1.00000 |
| SOL | 9 | 1,000,000,000 | 1.000000000 |

**Formula**: `Display Amount = Raw Amount / (10 ^ decimals)`

## 5. Sending SPL Tokens

### Creating Token Transfer

```typescript
import { 
  getAssociatedTokenAddress, 
  createTransferInstruction,
  getOrCreateAssociatedTokenAccount
} from '@solana/spl-token';
import { Transaction } from '@solana/web3.js';

async function sendSPLToken(
  connection: Connection,
  senderKeypair: Keypair,
  recipientAddress: string,
  tokenMintAddress: string,
  amount: number
) {
  const mintPublicKey = new PublicKey(tokenMintAddress);
  const recipientPublicKey = new PublicKey(recipientAddress);
  
  // Get mint info for decimals
  const mintInfo = await getMint(connection, mintPublicKey);
  
  // Convert amount to raw units
  const rawAmount = amount * Math.pow(10, mintInfo.decimals);
  
  // Get sender's ATA
  const senderATA = await getAssociatedTokenAddress(
    mintPublicKey,
    senderKeypair.publicKey
  );
  
  // Get or create recipient's ATA
  const recipientATA = await getAssociatedTokenAddress(
    mintPublicKey,
    recipientPublicKey
  );
  
  // Check if recipient ATA exists
  const recipientAccountInfo = await connection.getAccountInfo(recipientATA);
  
  const transaction = new Transaction();
  
  // If recipient ATA doesn't exist, create it
  if (!recipientAccountInfo) {
    const createATAIx = createAssociatedTokenAccountInstruction(
      senderKeypair.publicKey,  // Payer
      recipientATA,              // ATA address
      recipientPublicKey,        // Owner
      mintPublicKey              // Mint
    );
    transaction.add(createATAIx);
  }
  
  // Add transfer instruction
  const transferIx = createTransferInstruction(
    senderATA,           // Source
    recipientATA,        // Destination
    senderKeypair.publicKey,  // Owner
    rawAmount            // Amount
  );
  
  transaction.add(transferIx);
  
  // Send transaction
  const signature = await sendAndConfirmTransaction(
    connection,
    transaction,
    [senderKeypair]
  );
  
  return signature;
}
```

### Important Notes

⚠️ **Recipient ATA Creation Cost**: If the recipient doesn't have an ATA for that token, creating it costs ~0.002 SOL (rent). The sender pays this!

## 6. Jupiter Aggregator Integration

### What is Jupiter?

**Jupiter** is a DEX aggregator on Solana that finds the best swap routes across multiple decentralized exchanges.

**Real-Life Analogy**: Jupiter is like **Google Flights for crypto swaps**. Instead of checking prices on every exchange manually, Jupiter checks them all and shows you the best deal.

### How Jupiter Works

```
┌─────────────┐
│   User      │ "Swap 1 SOL for USDC"
│   Request   │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────┐
│      Jupiter Aggregator             │
│  Checks: Orca, Raydium, Lifinity,  │
│          Meteora, Phoenix, etc.     │
└──────┬──────────────────────────────┘
       │
       ▼
┌─────────────┐
│  Best Route │ Orca: 1 SOL → 195.32 USDC ✅
│   Found     │ Raydium: 1 SOL → 194.87 USDC
│             │ Meteora: 1 SOL → 195.01 USDC
└─────────────┘
```

### Jupiter API Endpoints

| Endpoint | Purpose |
|----------|---------|
| `/quote` | Get swap quote (prices, routes) |
| `/swap` | Get swap transaction |
| `/swap-instructions` | Get swap instructions for custom tx |
| `/tokens` | List all tradeable tokens |
| `/price` | Get current token prices |

## 8. Executing Swaps

### Getting Swap Transaction

```typescript
async function executeSwap(
  connection: Connection,
  userKeypair: Keypair,
  quote: any
) {
  // Step 1: Get swap transaction from Jupiter
  const swapResponse = await fetch('https://quote-api.jup.ag/v6/swap', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      quoteResponse: quote,
      userPublicKey: userKeypair.publicKey.toString(),
      wrapAndUnwrapSol: true,  // Auto wrap/unwrap SOL
      dynamicComputeUnitLimit: true,  // Optimize compute units
      prioritizationFeeLamports: 'auto'  // Auto priority fee
    })
  });
  
  const { swapTransaction } = await swapResponse.json();
  
  // Step 2: Deserialize the transaction
  const transactionBuf = Buffer.from(swapTransaction, 'base64');
  const transaction = VersionedTransaction.deserialize(transactionBuf);
  
  // Step 3: Sign the transaction
  transaction.sign([userKeypair]);
  
  // Step 4: Send transaction
  const rawTransaction = transaction.serialize();
  const txid = await connection.sendRawTransaction(rawTransaction, {
    skipPreflight: true,
    maxRetries: 2
  });
  
  // Step 5: Confirm transaction
  await connection.confirmTransaction(txid, 'confirmed');
  
  console.log(`Swap successful! Transaction: ${txid}`);
  
  return txid;
}
```

### Price Checking Service

```typescript
interface PriceAlert {
  id: number;
  telegramUserId: number;
  tokenMint: string;
  condition: 'above' | 'below';
  targetPrice: number;
  triggered: boolean;
}

class PriceAlertService {
  private bot: Telegraf;
  private alerts: Map<string, PriceAlert[]> = new Map();
  
  constructor(bot: Telegraf) {
    this.bot = bot;
    this.startPriceMonitoring();
  }
  
  // Add new alert
  addAlert(alert: PriceAlert) {
    const key = alert.tokenMint;
    if (!this.alerts.has(key)) {
      this.alerts.set(key, []);
    }
    this.alerts.get(key)!.push(alert);
  }
  
  // Check prices every minute
  private startPriceMonitoring() {
    setInterval(async () => {
      for (const [tokenMint, alerts] of this.alerts) {
        const currentPrice = await this.fetchTokenPrice(tokenMint);
        
        for (const alert of alerts) {
          if (alert.triggered) continue;
          
          const shouldTrigger = 
            (alert.condition === 'above' && currentPrice >= alert.targetPrice) ||
            (alert.condition === 'below' && currentPrice <= alert.targetPrice);
          
          if (shouldTrigger) {
            await this.triggerAlert(alert, currentPrice);
          }
        }
      }
    }, 60000); // Check every minute
  }
  
  // Fetch current price from Jupiter
  private async fetchTokenPrice(tokenMint: string): Promise<number> {
    const response = await fetch(
      `https://price.jup.ag/v4/price?ids=${tokenMint}`
    );
    const data = await response.json();
    return data.data[tokenMint]?.price || 0;
  }
  
  // Trigger alert notification
  private async triggerAlert(alert: PriceAlert, currentPrice: number) {
    alert.triggered = true;
    
    const message = `
🚨 **Price Alert Triggered!**

Token: ${this.getTokenSymbol(alert.tokenMint)}
Condition: ${alert.condition}
Target Price: $${alert.targetPrice}
Current Price: $${currentPrice}

Time to take action! 🚀
`;
    
    await this.bot.telegram.sendMessage(
      alert.telegramUserId,
      message,
      { parse_mode: 'Markdown' }
    );
  }
  
  private getTokenSymbol(mint: string): string {
    const symbols: Record<string, string> = {
      'EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v': 'USDC',
      'So11111111111111111111111111111111111111112': 'SOL',
      'DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263': 'BONK',
    };
    return symbols[mint] || 'Unknown';
  }
}
```

## 10. Portfolio Tracking

### Fetching All Token Holdings

```typescript
async function getUserPortfolio(
  connection: Connection,
  walletPublicKey: PublicKey
): Promise<TokenHolding[]> {
  // Get all token accounts owned by the wallet
  const tokenAccounts = await connection.getParsedTokenAccountsByOwner(
    walletPublicKey,
    { programId: TOKEN_PROGRAM_ID }
  );
  
  const holdings: TokenHolding[] = [];
  
  for (const { account } of tokenAccounts.value) {
    const parsedInfo = account.data.parsed.info;
    
    // Skip empty accounts
    if (parsedInfo.tokenAmount.uiAmount === 0) continue;
    
    const mint = parsedInfo.mint;
    const amount = parsedInfo.tokenAmount.uiAmount;
    
    // Fetch token metadata
    const metadata = await fetchTokenMetadata(mint);
    
    // Fetch current price
    const price = await fetchTokenPrice(mint);
    
    holdings.push({
      mint,
      symbol: metadata.symbol,
      name: metadata.name,
      amount,
      decimals: parsedInfo.tokenAmount.decimals,
      price,
      value: amount * price
    });
  }
  
  // Sort by value (highest first)
  holdings.sort((a, b) => b.value - a.value);
  
  return holdings;
}

interface TokenHolding {
  mint: string;
  symbol: string;
  name: string;
  amount: number;
  decimals: number;
  price: number;
  value: number;
}
```

## 11. Transaction Simulation

### Why Simulate?

**Transaction simulation** lets you test a transaction before actually sending it. This helps:
- Detect errors before spending fees
- Estimate compute units needed
- Check if transaction will succeed

**Real-Life Analogy**: Like a **test drive before buying a car**. You want to know if it works before committing.

### Simulating Transactions

```typescript
async function simulateTransaction(
  connection: Connection,
  transaction: Transaction,
  signers: Keypair[]
): Promise<{ success: boolean; error?: string; logs?: string[] }> {
  try {
    // Get recent blockhash
    const { blockhash } = await connection.getLatestBlockhash();
    transaction.recentBlockhash = blockhash;
    transaction.feePayer = signers[0].publicKey;
    
    // Sign transaction
    transaction.sign(...signers);
    
    // Simulate
    const simulation = await connection.simulateTransaction(transaction);
    
    if (simulation.value.err) {
      return {
        success: false,
        error: JSON.stringify(simulation.value.err),
        logs: simulation.value.logs || []
      };
    }
    
    return {
      success: true,
      logs: simulation.value.logs || []
    };
    
  } catch (error: any) {
    return {
      success: false,
      error: error.message
    };
  }
}

// Usage before sending
const simulation = await simulateTransaction(connection, transaction, [userKeypair]);

if (!simulation.success) {
  console.error('Transaction will fail:', simulation.error);
  console.log('Logs:', simulation.logs);
  // Don't send the transaction
} else {
  console.log('Transaction should succeed!');
  // Proceed with sending
}
```

## 12. Database Integration with PostgreSQL

### Why Use a Database?

In-memory storage (like `Record<string, Keypair>`) loses all data when the bot restarts. A database provides:
- **Persistence** - Data survives restarts
- **Scalability** - Handle millions of users
- **Queries** - Complex searches and analytics
- **Backup** - Disaster recovery

### Encrypting Private Keys

```typescript
import crypto from 'crypto';

const ENCRYPTION_KEY = process.env.ENCRYPTION_KEY!; // 32-byte key
const ALGORITHM = 'aes-256-gcm';

function encryptPrivateKey(privateKey: Uint8Array): string {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv(ALGORITHM, ENCRYPTION_KEY, iv);
  
  const encrypted = Buffer.concat([
    cipher.update(Buffer.from(privateKey)),
    cipher.final()
  ]);
  
  const authTag = cipher.getAuthTag();
  
  // Combine iv + authTag + encrypted
  const combined = Buffer.concat([iv, authTag, encrypted]);
  
  return combined.toString('base64');
}

function decryptPrivateKey(encryptedData: string): Uint8Array {
  const combined = Buffer.from(encryptedData, 'base64');
  
  const iv = combined.slice(0, 16);
  const authTag = combined.slice(16, 32);
  const encrypted = combined.slice(32);
  
  const decipher = crypto.createDecipheriv(ALGORITHM, ENCRYPTION_KEY, iv);
  decipher.setAuthTag(authTag);
  
  const decrypted = Buffer.concat([
    decipher.update(encrypted),
    decipher.final()
  ]);
  
  return new Uint8Array(decrypted);
}
```

## 13. Revenue Model Implementation

### How Trading Bots Make Money

| Revenue Source | Description | Typical Fee |

| **Swap Fees** | Small % on every swap | 0.5% - 1% |

### Adding Fees to Swaps

```typescript
async function executeSwapWithFee(

  feePercentage: number = 0.5  // 0.5% fee

  const FEE_WALLET = new PublicKey(process.env.FEE_WALLET_ADDRESS!);
  
  // Calculate fee in output token
  const outputAmount = Number(quote.outAmount);
  const feeAmount = Math.floor(outputAmount * (feePercentage / 100));
  const userAmount = outputAmount - feeAmount;
  
  console.log(`User receives: ${userAmount}`);
  console.log(`Platform fee: ${feeAmount}`);
  
  // Get swap transaction
  const swapResponse = await fetch('https://quote-api.jup.ag/v6/swap', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      quoteResponse: quote,
      userPublicKey: userKeypair.publicKey.toString(),
      wrapAndUnwrapSol: true,
      feeAccount: FEE_WALLET.toString()  // Jupiter's fee account parameter
    })
  });

}
```

### Tracking Revenue

```sql
CREATE TABLE revenue_tracking (
  id SERIAL PRIMARY KEY,
  transaction_signature VARCHAR(88) NOT NULL,
  user_telegram_id BIGINT,
  fee_amount BIGINT NOT NULL,
  fee_token_mint VARCHAR(44) NOT NULL,
  usd_value DECIMAL(20, 2),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Query total revenue
SELECT 
  token_mint,
  SUM(fee_amount) as total_fees,
  SUM(usd_value) as total_usd
FROM revenue_tracking
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY token_mint;
```
