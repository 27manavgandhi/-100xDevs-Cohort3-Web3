## 1. What are PDAs?

**Program Derived Addresses (PDAs)** are special addresses in Solana that:
- Have **no private key**
- Are **deterministically derived** from seeds
- Can only be **signed by programs**
- Live **off the ed25519 curve**

### Real-Life Analogy

Think of PDAs like a **company bank account**:
- **Regular Account** = Personal bank account (you have the key/password)
- **PDA** = Company account (only the company can sign, not individuals)

### Why PDAs?

In Ethereum, contracts have addresses and can hold assets. In Solana:
- **Programs are stateless** (just code)
- **Accounts hold data** (state)
- **PDAs let programs "own" accounts**

```
Ethereum:                    Solana:
┌────────────────┐          ┌────────────────┐
│ Smart Contract │          │ Program (code) │
│ - Has address  │          │ - Has address  │
│ - Holds ETH    │          │ - No storage   │
│ - Has storage  │          └────────────────┘
└────────────────┘                   │ owns
                                     ↓
                             ┌────────────────┐
                             │ PDA Account    │
                             │ - Has address  │
                             │ - Holds SOL    │
                             │ - Has storage  │
                             └────────────────┘
```

## 2. How PDAs Work

### The Ed25519 Curve

Normal Solana addresses are **public keys on the ed25519 curve**:
```
Private Key → Public Key (on curve) → Address
```

PDAs are **off the curve**:
```
Seeds + Program ID → Hash → Address (off curve)
                              ↓
                         No private key!
```

### Finding a PDA

Not all hash outputs are off the curve. We use a **bump seed**:

```rust
fn find_program_address(
    seeds: &[&[u8]], 
    program_id: &Pubkey
) -> (Pubkey, u8)
```

**Algorithm:**
1. Start with bump = 255
2. Hash (seeds + [bump] + program_id)
3. Is result off curve? ✅ Return (address, bump)
4. If on curve, try bump = 254, 253, 252...
5. Repeat until off-curve address found

```
seeds: ["vault"]
program_id: "11111..."

Try bump 255: Hash → On curve ❌
Try bump 254: Hash → On curve ❌
Try bump 253: Hash → Off curve ✅ Found PDA!

PDA = Hash("vault" + [253] + "11111...")
Canonical bump = 253
```

## 3. Creating Your First PDA

### Contract Side (Rust)

```rust
use solana_program::{
    account_info::AccountInfo,
    entrypoint,
    entrypoint::ProgramResult,
    msg,
    pubkey::Pubkey,
};

entrypoint!(process_instruction);

pub fn process_instruction(
    program_id: &Pubkey,
    _accounts: &[AccountInfo],
    _instruction_data: &[u8],
) -> ProgramResult {
    msg!("Hello, world!");
    
    // Find PDA
    let (pda, bump) = Pubkey::find_program_address(
        &[b"vault"],
        program_id
    );
    
    msg!("PDA: {}", pda);
    msg!("Bump: {}", bump);
    
    Ok(())
}
```

### Key Points

- **Seeds**: Array of byte slices (`&[&[u8]]`)
- **Common seeds**: strings, pubkeys, numbers
- **Returns**: (PDA address, bump seed)
- **Deterministic**: Same seeds → same PDA always

```typescript
import { Connection, Keypair, PublicKey, Transaction, SystemProgram, sendAndConfirmTransaction } from "@solana/web3.js";
import fs from "fs";

const connection = new Connection("https://api.devnet.solana.com");

// Load keypair from file
const keypairPath = "/path/to/your/keypair.json";
const secretKeyString = fs.readFileSync(keypairPath, 'utf8');
const secretKey = Uint8Array.from(JSON.parse(secretKeyString));
const payer = Keypair.fromSecretKey(secretKey);

// Your program ID (from deployment)
const programId = new PublicKey("YOUR_PROGRAM_ID_HERE");

async function main() {
    console.log("Calling program...");
    
    // Create transaction
    const tx = new Transaction();
    
    // Add instruction
    tx.add({
        keys: [],
        programId: programId,
        data: Buffer.alloc(0), // Empty data
    });
    
    // Send transaction
    const signature = await sendAndConfirmTransaction(
        connection,
        tx,
        [payer]
    );
    
    console.log("Transaction signature:", signature);
    console.log("View on explorer:");
    console.log(`https://explorer.solana.com/tx/${signature}?cluster=devnet`);
}

main();
```

## 6. Cross-Program Invocation (CPI)

Programs can call other programs using CPI.

### Regular invoke

```rust
use solana_program::program::invoke;

invoke(
    &instruction,
    &[account1, account2, account3],
)?;
```

### invoke_signed (for PDAs)

```rust
use solana_program::program::invoke_signed;

invoke_signed(
    &instruction,
    &[account1, account2, account3],
    &[&[b"vault", &[bump]]], // PDA seeds + bump
)?;
```

## 7. PDA Seeds Patterns

### Pattern 1: Global Singleton

```rust
// Single PDA for entire program
let (pda, bump) = Pubkey::find_program_address(
    &[b"config"],
    program_id
);
```

**Use case**: Global program configuration

### Pattern 2: User-Specific

```rust
// One PDA per user
let (pda, bump) = Pubkey::find_program_address(
    &[b"user", user_pubkey.as_ref()],
    program_id
);
```

**Use case**: User profiles, user token accounts

### Pattern 3: Hierarchical

```rust
// PDA with multiple seeds
let (pda, bump) = Pubkey::find_program_address(
    &[
        b"vault",
        mint_pubkey.as_ref(),
        authority.as_ref()
    ],
    program_id
);
```

**Use case**: Token vaults, escrow accounts

### Pattern 4: Indexed

```rust
// PDA with counter
let index: u64 = 5;
let (pda, bump) = Pubkey::find_program_address(
    &[b"item", &index.to_le_bytes()],
    program_id
);
```

**Use case**: NFT metadata, sequential records

## 8. PDA Ownership

### Who Owns a PDA Account?

```rust
// PDA address
let (pda, _bump) = Pubkey::find_program_address(
    &[b"vault"],
    program_id
);

// Create account at PDA
// The account's OWNER is the program
// The program can modify the account's data
```

**Key Insight:**
- PDA is just an **address**
- Account at PDA address has an **owner** (usually the program)
- Only owner program can modify account data

## 9. Security Considerations

### ⚠️ Critical PDA Security Rules

#### 1. Verify PDA Derivation

```rust
// ❌ UNSAFE: Trust user-provided PDA
let pda = accounts[0].key;

// ✅ SAFE: Derive and verify
let (expected_pda, bump) = Pubkey::find_program_address(
    &[b"vault"],
    program_id
);

if pda != expected_pda {
    return Err(ProgramError::InvalidAccountData);
}
```

#### 2. Store and Verify Bump

```rust
// ❌ UNSAFE: Recalculate bump every time (expensive)
let (_pda, bump) = Pubkey::find_program_address(...);

// ✅ SAFE: Store canonical bump in account data
#[derive(BorshSerialize, BorshDeserialize)]
struct VaultAccount {
    bump: u8,
    authority: Pubkey,
    balance: u64,
}
```

#### 3. Prevent Seed Collision

```rust
// ❌ UNSAFE: User-controlled seeds
let (pda, _) = Pubkey::find_program_address(
    &[user_input.as_bytes()], // Attacker controls!
    program_id
);

// ✅ SAFE: Combine with known values
let (pda, _) = Pubkey::find_program_address(
    &[b"vault", user_pubkey.as_ref()],
    program_id
);
```

## 10. Real-World Use Cases

### Use Case 1: Token Vault

```rust
// PDA owns token account
// Program can transfer tokens via invoke_signed

let (vault_pda, bump) = Pubkey::find_program_address(
    &[b"token_vault", mint.as_ref()],
    program_id
);

// Later, when transferring tokens:
invoke_signed(
    &spl_token::instruction::transfer(
        token_program,
        vault_token_account,
        destination,
        vault_pda, // PDA is authority
        &[],
        amount,
    )?,
    &[...],
    &[&[b"token_vault", mint.as_ref(), &[bump]]]
)?;
```

### Use Case 2: Escrow

```rust
// PDA holds funds until conditions met
let (escrow_pda, bump) = Pubkey::find_program_address(
    &[b"escrow", seller.as_ref(), buyer.as_ref()],
    program_id
);
```

### Use Case 3: Associated Token Account

```rust
// Standard pattern in SPL Token
let (ata, _) = Pubkey::find_program_address(
    &[
        wallet.as_ref(),
        token_program.as_ref(),
        mint.as_ref(),
    ],
    &spl_associated_token_account::id()
);
```

## 11. Complete PDA Example

### Full Contract with PDA

```rust
use solana_program::{
    account_info::{next_account_info, AccountInfo},
    entrypoint,
    entrypoint::ProgramResult,
    msg,
    program_error::ProgramError,
    pubkey::Pubkey,
    system_instruction,
    program::invoke_signed,
    rent::Rent,
    sysvar::Sysvar,
};
use borsh::{BorshSerialize, BorshDeserialize};

#[derive(BorshSerialize, BorshDeserialize, Debug)]
pub struct VaultAccount {
    pub bump: u8,
    pub authority: Pubkey,
    pub balance: u64,
}

entrypoint!(process_instruction);

pub fn process_instruction(
    program_id: &Pubkey,
    accounts: &[AccountInfo],
    instruction_data: &[u8],
) -> ProgramResult {
    let accounts_iter = &mut accounts.iter();
    
    // Account 0: PDA account (to be created)
    let pda_account = next_account_info(accounts_iter)?;
    
    // Account 1: User/authority
    let authority = next_account_info(accounts_iter)?;
    
    // Account 2: System program
    let system_program = next_account_info(accounts_iter)?;
    
    // Derive PDA
    let (expected_pda, bump) = Pubkey::find_program_address(
        &[b"vault", authority.key.as_ref()],
        program_id
    );
    
    // Verify PDA
    if pda_account.key != &expected_pda {
        msg!("Invalid PDA");
        return Err(ProgramError::InvalidAccountData);
    }
    
    msg!("PDA verified: {}", expected_pda);
    msg!("Bump: {}", bump);
    
    // Create account at PDA if it doesn't exist
    if pda_account.lamports() == 0 {
        let space = 8 + 1 + 32 + 8; // discriminator + bump + pubkey + u64
        let rent = Rent::get()?;
        let lamports = rent.minimum_balance(space);
        
        invoke_signed(
            &system_instruction::create_account(
                authority.key,
                pda_account.key,
                lamports,
                space as u64,
                program_id,
            ),
            &[authority.clone(), pda_account.clone(), system_program.clone()],
            &[&[b"vault", authority.key.as_ref(), &[bump]]]
        )?;
        
        // Initialize account data
        let vault_data = VaultAccount {
            bump,
            authority: *authority.key,
            balance: 0,
        };
        
        vault_data.serialize(&mut &mut pda_account.data.borrow_mut()[..])?;
        
        msg!("PDA account created!");
    } else {
        msg!("PDA account already exists");
    }
    
    Ok(())
}
```

## 12. Testing PDAs

### Test Script

```typescript
import { 
    Connection, 
    Keypair, 
    PublicKey, 
    Transaction,
    TransactionInstruction,
    SystemProgram,
    sendAndConfirmTransaction 
} from "@solana/web3.js";

const connection = new Connection("https://api.devnet.solana.com");
const programId = new PublicKey("YOUR_PROGRAM_ID");
const payer = Keypair.fromSecretKey(/* your secret key */);

async function testPDA() {
    // Derive PDA on client side
    const [pda, bump] = PublicKey.findProgramAddressSync(
        [Buffer.from("vault"), payer.publicKey.toBuffer()],
        programId
    );
    
    console.log("PDA:", pda.toBase58());
    console.log("Bump:", bump);
    
    // Create instruction
    const ix = new TransactionInstruction({
        keys: [
            { pubkey: pda, isSigner: false, isWritable: true },
            { pubkey: payer.publicKey, isSigner: true, isWritable: true },
            { pubkey: SystemProgram.programId, isSigner: false, isWritable: false },
        ],
        programId,
        data: Buffer.alloc(0),
    });
    
    // Send transaction
    const tx = new Transaction().add(ix);
    const sig = await sendAndConfirmTransaction(connection, tx, [payer]);
    
    console.log("Transaction:", sig);
}

testPDA();
```

## Key Concepts Table

| Concept | Description |
|---------|-------------|
| **PDA** | Program Derived Address - address without private key |
| **Bump** | Number (0-255) to find off-curve address |
| **Canonical Bump** | First bump that produces valid PDA (usually 254-255) |
| **Seeds** | Inputs used to derive PDA |
| **invoke_signed** | CPI function that signs with PDA |
| **On-curve** | Valid ed25519 public key (has private key) |
| **Off-curve** | Invalid public key (no private key - PDA) |
| **CPI** | Cross-Program Invocation |
| **Authority** | Account that controls a PDA |
| **Signer Seeds** | Seeds + bump used in invoke_signed |
