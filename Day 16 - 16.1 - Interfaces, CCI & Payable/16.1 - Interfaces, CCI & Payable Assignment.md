## Task 5 — Will Contract (Main Assignment)

Create a Will Contract as specified in the class assignment.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract WillContract {
  // TODO: State variables
  //   - uint lastVisited (timestamp of last owner interaction)
  //   - uint inactivityPeriod (1 year in seconds)
  //   - address owner
  //   - address recipient
  
  // TODO: Constructor(address _recipient)
  //   - Set owner = msg.sender
  //   - Set recipient = _recipient
  //   - Set lastVisited = block.timestamp
  //   - Set inactivityPeriod = 365 days (or 365 * 24 * 60 * 60)
  
  // TODO: Modifier onlyOwner
  //   - Require msg.sender == owner
  
  // TODO: Modifier onlyRecipient
  //   - Require msg.sender == recipient
  
  // TODO: Function changeRecipient(address newRecipient) onlyOwner
  //   - Update recipient
  //   - Update lastVisited
  
  // TODO: Function deposit() payable onlyOwner
  //   - Accept Ether (no additional code needed if payable)
  //   - Update lastVisited = block.timestamp
  
  // TODO: Function ping() onlyOwner
  //   - Update lastVisited = block.timestamp
  
  // TODO: Function claim() onlyRecipient
  //   - Require: block.timestamp >= lastVisited + inactivityPeriod
  //   - Transfer all Ether to recipient
  //   - Use: payable(recipient).transfer(address(this).balance)
  
  // TODO: Function getBalance() view returns (uint)
  //   - Return address(this).balance
  
  // TODO: Function getTimeSinceLastVisit() view returns (uint)
  //   - Return block.timestamp - lastVisited
  
  // TODO: Function canClaim() view returns (bool)
  //   - Return whether recipient can claim (inactivity period passed)
}
```

**Requirements:**
- Implement all modifiers correctly
- Use block.timestamp for time tracking
- Use 365 days for inactivity period (Solidity time units)
- Recipient can only claim after 1 year of owner inactivity
- Test all functions: deposit, ping, changeRecipient, claim

## Conceptual Questions

1. What is the difference between an **interface** and an **abstract contract**? When would you use each?

2. Why can't interfaces have state variables or constructors? What is the purpose of this restriction?

3. Explain the **msg.value** and **msg.sender** global variables. How are they used in payable functions?

4. What is **Cross-Contract Invocation (CCI)**? Give three real-world examples of when you'd use it in a dApp.

5. Why do we need to use an interface to call functions in another contract? Can't we just call them directly?

6. What happens if you try to send Ether to a function that is not marked as `payable`? How does Solidity prevent this?

7. Explain the difference between `transfer()`, `send()`, and `call()` for sending Ether. Which is recommended in modern Solidity?

8. In the Will Contract, why do we use `block.timestamp` instead of maintaining a counter? What are the security considerations with `block.timestamp`?

9. What is the purpose of the **modifier** keyword in Solidity? How does it improve code reusability?

10. Explain how the **require** statement works in the Will Contract's `claim()` function. What happens if the condition fails?

## Testing Guide

### Testing in Remix IDE

1. **Deploy contracts**
   - Start with base contracts (e.g., Counter, SimpleToken)
   - Then deploy contracts that interact with them

2. **Test CCI**
   - Copy deployed contract address
   - Pass it to functions that do CCI
   - Verify state changes in both contracts

3. **Test Payable**
   - Set VALUE field before calling payable functions
   - Check balances using getBalance functions
   - Verify Ether transfers correctly

4. **Test Time-based Logic** (Will Contract)
   - Since you can't wait 1 year, modify inactivityPeriod for testing
   - Example: Use `1 minutes` instead of `365 days`
   - Test ping resets the timer
   - Test claim works after period expires

### Common Pitfalls

- ❌ Forgetting to mark functions `external` in interfaces
- ❌ Not using `payable` when receiving Ether
- ❌ Incorrect units (wei vs ether) when dealing with msg.value
- ❌ Not checking balance before transfer
- ❌ Forgetting to update lastVisited in Will Contract
- ❌ Using `address` instead of `address payable` for transfers
