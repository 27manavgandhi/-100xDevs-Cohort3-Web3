## Core Assignment

### Task 1 — Complete the Token Launchpad

Build a fully functional **Solana Token Launchpad** where a user can:

1. Connect their Phantom / Backpack wallet
2. Fill in token details:
   - **Name** (e.g., "MyToken")
   - **Symbol** (e.g., "MTK")
   - **Image URL** (points to token logo)
   - **Initial Supply** (e.g., 1000000)
3. Click "Create Token" → wallet approval popup → token created on devnet
4. Show the new **Mint Address** after creation

**Tech Stack:**
```bash
npm create vite@latest token-launchpad -- --template react
cd token-launchpad
npm install @solana/wallet-adapter-base @solana/wallet-adapter-react \
            @solana/wallet-adapter-react-ui @solana/wallet-adapter-wallets \
            @solana/web3.js @solana/spl-token
npm install --save-dev vite-plugin-node-polyfills
```

The transaction must include all 4 instructions in one go:
1. `SystemProgram.createAccount` — allocate the mint account
2. `createInitializeMintInstruction` — set decimals + authorities
3. `createAssociatedTokenAccountInstruction` — create ATA for creator
4. `createMintToInstruction` — mint initial supply to creator's ATA

### Task 2 — Add Token-22 with Metadata

Upgrade the launchpad to use **Token-22** with on-chain metadata so the token shows up with a name, symbol, and image in Phantom wallet.

Additional instructions needed:
- `createInitializeMetadataPointerInstruction`

After minting, the token should appear in Phantom with your custom name and logo.

## Bonus Challenges

### Challenge 1 — Add Freeze Authority Toggle

Add a checkbox "Enable Freeze Authority". If unchecked, pass `null` as `freezeAuthority` in `createInitializeMintInstruction`. This makes the token non-freezable — a common requirement for decentralized tokens.

### Challenge 2 — Add Decimals Input

Instead of hardcoding `9` decimals, let the user choose (0–9). Note: 0 decimals = whole numbers only (good for NFT-like tokens).

### Challenge 3 — Add Revoke Mint Authority

After creating the token, add a "Revoke Mint Authority" button that calls `setAuthority` to set the mint authority to `null`. This permanently caps the token supply — commonly done to increase trust.

```js
import { setAuthority, AuthorityType } from '@solana/spl-token';

await setAuthority(
  connection,
  wallet,
  mintAddress,
  wallet.publicKey,
  AuthorityType.MintTokens,
  null  // setting to null revokes permanently
);
```
