## Core Assignment — Build a Centralized LST Server

### Overview

Build a Node.js/TypeScript server that acts as a **centralized Liquid Staking Token** system:

- When SOL arrives at your address → **mint** custom tokens proportional to the SOL amount
- When custom tokens arrive at your address → **burn** them and **send SOL back**

This simulates how a real LST protocol works (Marinade, Lido) but in a centralized way.

```
User ──sends SOL──► Your Address ──► Indexer (Helius webhook)
                                         │
                                         ▼
                                    Node.js server
                                    if (received >= 2 SOL) {
                                      mint custom tokens to sender
                                    }
                                    if (received custom tokens) {
                                      burn tokens
                                      send SOL back to sender
                                    }
```

## Bonus Challenges

### Challenge 1 — Store Events in a Database

Add a SQLite/PostgreSQL database to:
- Log every incoming webhook event with its signature
- Before minting, check if the signature was already processed
- This prevents double-minting from duplicate webhook deliveries

### Challenge 2 — Dynamic Exchange Rate

Instead of a fixed 1:1 ratio, implement a dynamic exchange rate:
- Track total SOL held and total tokens minted
- Exchange rate = totalSOL / totalTokensMinted
- As you earn yield (staking rewards), the rate improves over time
