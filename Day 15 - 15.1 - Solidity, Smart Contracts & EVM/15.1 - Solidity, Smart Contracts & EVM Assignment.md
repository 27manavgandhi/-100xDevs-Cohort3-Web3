## Conceptual Questions

1. What is the difference between **bytecode** and **ABI**? Why do we need both?

2. Explain the compilation flow from Solidity code to execution on the EVM. What happens at each step?

3. What is the difference between a **state-changing function** and a **view function**? When would you use each, and why does one cost gas while the other doesn't (when called externally)?

4. What is the purpose of the **constructor** in Solidity? When is it executed, and can you call it again after deployment?

5. Explain the four visibility modifiers in Solidity: `public`, `external`, `internal`, and `private`. Give an example use case for each.

6. What does the `virtual` keyword do in a parent contract function? What is the `override` keyword used for in a child contract?

7. Why is the `require` statement important in smart contracts? What happens when a `require` condition fails?

8. What is a **mapping** in Solidity? How is it different from an array, and when would you use one over the other?

9. What are **structs** in Solidity? Give a real-world example where you would use a struct instead of multiple separate variables.

10. Explain the pragma directive `pragma solidity ^0.8.0;`. What does the `^` symbol mean, and why is it important to specify a Solidity version?

## Deployment & Testing Guide

### Using Remix IDE

1. **Open Remix:** Go to https://remix.ethereum.org/
2. **Create new file:** Click "+" icon in File Explorer, name it (e.g., `Calculator.sol`)
3. **Write contract:** Paste your Solidity code
4. **Compile:** Click "Solidity Compiler" tab, click "Compile"
5. **Deploy:** Click "Deploy & Run Transactions" tab
   - Select "Remix VM (Shanghai)" environment
   - Click "Deploy"
6. **Interact:** Use the deployed contract interface to call functions
7. **Check ABI & Bytecode:** In compiler tab, click "Compilation Details" to see ABI and bytecode

### Testing Checklist

For each contract:
- ✅ Compiles without errors
- ✅ Constructor works correctly
- ✅ State-changing functions update storage
- ✅ View functions return correct data
- ✅ Require statements prevent invalid operations
- ✅ Events are emitted (check transaction logs)
- ✅ Inheritance works (child can access parent functions)
- ✅ Test edge cases (zero values, unauthorized access, etc.)
