## 1. Why Rust for Solana?

Solana uses **Rust** (not Solidity) for writing smart contracts (called "programs").

### Rust vs Solidity

| Feature | Solidity (Ethereum) | Rust (Solana) |
|---------|---------------------|---------------|
| **Memory Safety** | Managed by EVM | Compile-time guarantees |
| **Performance** | Interpreted | Compiled to native code |
| **Concurrency** | Sequential | Parallel execution |
| **Type System** | Simple | Advanced with traits |
| **Learning Curve** | Moderate | Steep |

### Real-Life Analogy

Think of Rust vs Solidity like:
- **Solidity** = Automatic car (easy to drive, some control)
- **Rust** = Manual car (harder to learn, full control, better performance)

## 5. Mutability

**By default, all variables in Rust are immutable.**

### This Won't Compile:

```rust
fn main() {
    let x = 5;
    println!("The value of x is: {x}");
    x = 6;  // ❌ ERROR: cannot assign twice to immutable variable
    println!("The value of x is: {x}");
}
```

### This Will Compile:

```rust
fn main() {
    let mut x = 5;
    println!("The value of x is: {x}");
    x = 6;  // ✅ OK: x is mutable
    println!("The value of x is: {x}");
}
```

### Why Immutability by Default?

- **Safety**: Prevents accidental mutations
- **Concurrency**: Easier to reason about in parallel code
- **Performance**: Compiler can optimize better

## 7. Ownership - The Core of Rust

Ownership is Rust's most unique feature. It enables memory safety without garbage collection.

### The Problem

```rust
fn main() {
    let str = String::from("Harkirat");
    let len = get_length(str);
    println!("{}", len);

    print!("{}", str);  // ❌ ERROR: value borrowed here after move
}

fn get_length(str: String) -> usize {
    return str.len()
}
```

### Why Does This Fail?

```
main() creates str → Ownership of str
       ↓
get_length(str) → Ownership MOVES to get_length
       ↓
Function ends → str is DROPPED (memory freed)
       ↓
print!("{}", str) → ❌ str no longer exists!
```

### Stack vs Heap

```
Stack (Fast, Fixed Size)        Heap (Slower, Dynamic Size)
┌─────────────────────┐         ┌──────────────────────┐
│ i32: 42             │         │ String: "Harkirat"   │
│ bool: true          │         │ Vec: [1, 2, 3]       │
│ &str: pointer ─────────────>  │ (actual data here)   │
└─────────────────────┘         └──────────────────────┘
```

**Stack Types** (Copy): i32, bool, f64, &str
**Heap Types** (Move): String, Vec, custom structs with heap data

---

## 8. Ownership Rules

1. **Each value has an owner**
2. **There can only be ONE owner at a time**
3. **When the owner goes out of scope, the value is dropped**

### Fix #1 - Return Ownership

```rust
fn main() {
    let str = String::from("Harkirat");
    let (str, len) = get_length(str);  // Get ownership back!
    println!("{} {}", str, len);
}

fn get_length(str: String) -> (String, usize) {
    let len = str.len();
    return (str, len);  // Return ownership
}
```

## 9. Borrowing & References

Rather than transferring ownership, you can **borrow** a reference.

```rust
fn main() {
    let str = String::from("Harkirat");
    let len = get_length(&str);  // Borrow with &
    println!("{} {}", str, len);  // ✅ str still valid!
}

fn get_length(str: &String) -> usize {
    let len = str.len();
    return len
}
```

### How Borrowing Works

```
main() owns str
    ↓
get_length(&str) borrows str
    ↓
Function reads str (doesn't own it)
    ↓
Function ends, borrow ends
    ↓
main() still owns str ✅
```

### Borrowing Rules

1. **At any time**, you can have **either**:
   - ONE mutable reference, **OR**
   - ANY number of immutable references
2. **References must always be valid**

### Quiz 1: Will This Compile?

```rust
fn main() {
    let str = String::from("Harkirat");
    let ref1 = &str;
    let ref2 = &str;

    println!("{} {}", ref1, ref2);
}
```

**Answer**: ✅ **YES** - Multiple immutable references are allowed!

### Quiz 2: Will This Compile?

```rust
fn main() {
    let mut str = String::from("Harkirat");
    let ref1 = &mut str;
    let ref2 = &str;
    
    println!("{} {}", ref1, ref2);
}
```

**Answer**: ❌ **NO** - Cannot have mutable AND immutable references at the same time!

### Quiz 3: Will This Compile?

```rust
fn main() {
    let mut str = String::from("Harkirat");
    let ref1 = &mut str;
    ref1.push_str("Singh");
    let ref2 = &str;
    
    println!("{}", ref2);
}
```

**Answer**: ✅ **YES** - ref1 is no longer used after push_str, so ref2 can borrow!

## 10. Structs

Structs let you structure data together (like objects in JavaScript).

```rust
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}

fn main() {
    let user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };
    print!("User 1 username: {}", user1.username);
}
```

### Stack vs Heap in Structs

```rust
struct User {
    active: bool,           // Stack (1 byte)
    sign_in_count: u64,     // Stack (8 bytes)
    username: String,       // Heap (pointer on stack)
}
```

### Copy Trait (Stack-Only Structs)

```rust
#[derive(Copy, Clone)]
struct User {
    active: bool,
    sign_in_count: u64,
}

fn main() {
    let user1 = User {
        active: true,
        sign_in_count: 1,
    };

    print_name(user1);  // user1 is COPIED
    print!("User 1 active: {}", user1.active);  // ✅ Still works!
}

fn print_name(user1: User) {
    print!("User 1 active: {}", user1.active);
}
```

### Clone Trait (Heap Structs)

```rust
#[derive(Clone)]
struct User {
    active: bool,
    sign_in_count: u64,
    username: String,  // Heap type - can't use Copy!
}

fn main() {
    let user1 = User {
        active: true,
        sign_in_count: 1,
        username: "harkirat".to_string()
    };

    change_name(user1.clone());  // Explicit clone
    print!("User 1 active: {}", user1.active);
}

fn change_name(user1: User) {
    print!("User 1 username: {}", user1.username);
}
```

## Key Concepts Table

| Concept | Description |
|---------|-------------|
| **Ownership** | Each value has exactly one owner |
| **Move** | Transfer ownership to another variable |
| **Borrow** | Temporary reference without ownership |
| **&** | Immutable reference |
| **&mut** | Mutable reference |
| **Copy trait** | Duplicate value (stack types only) |
| **Clone trait** | Deep copy value (including heap) |
| **Stack** | Fast, fixed-size memory |
| **Heap** | Slower, dynamic-size memory |
| **String** | Heap-allocated, growable string |
| **&str** | String slice, usually on stack |
| **Vec<T>** | Growable array on heap |
